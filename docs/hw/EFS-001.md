## 1. 목적

본 문서는 소형 챔버 온도제어 및 알람진단 시스템의 기능 요구사항, 시스템 아키텍처, 상태 머신, 통신 프로토콜을 정의한다.

본 프로젝트는 칠러 CS 엔지니어 포트폴리오를 목적으로 한다. PID 온도 제어, 2단계 알람 진단(WARNING / ALARM), CS 복구 절차를 실제 하드웨어에서 구현하여 실무 역량을 시연한다.

## 4. 하드웨어 구성 요약

| 항목 | 사양 |
|-----|------|
| MCU | STM32F446RE NUCLEO (ARM Cortex-M4, 180MHz) |
| 온도 센서 | DS18B20 (OneWire, ±0.5°C, 12-bit, 750ms/샘플) |
| 히터 | 카프톤 필름 히터 24V / ~22Ω / 약 26W |
| 히터 제어 | N-ch MOSFET 모듈 (PB0/A3, TIM3_CH3, 1kHz PWM) |
| 냉각 팬 | 5V DC 팬 + 릴레이 모듈 (PA0/A0) + 24V→5V DC-DC |
| 부저 | 액티브 부저 (PB10/D6, 3.3V) |
| 도어 스위치 | NO형 리미트 스위치 (PA1/A1, 내부 Pull-Up, 페일세이프) |
| 챔버 | 밀폐 용기 (~1L) |
| 전원 | WANPTEK 파워 서플라이 24V / 1.5A 제한 |

## 6. 상태 머신

### 6.1 상태 정의

| 상태 | 히터 | 팬 | 부저 | 진입 조건 |
|------|------|-----|------|---------|
| IDLE | OFF | OFF | OFF | 초기 또는 STOP/RESET 수신 |
| HEATING | PID PWM | 조건부 ON | OFF | EC `START` 수신 (IDLE) |
| WARNING | PID PWM | 조건부 ON | 간헐 | ALM-01 (TEMP_WARNING) 발생 |
| ALARM | OFF | ON | 연속 | ALM-02 (TEMP_ALARM) 또는 ALM-03 (SENSOR_ERROR) 발생 |

### 6.2 상태 전이 다이어그램

```
              START
  IDLE ─────────────────► HEATING
   ▲                          │
   │ RESET                    │ ALM-01 (TEMP_WARNING)
   │ (조건 충족)              ▼
   │                      WARNING ──────────────────── HEATING
   │                          │   온도 복귀 (자동 해제)
   │                          │ ALM-02 또는 ALM-03
   │                          ▼
   └──────────────────── ALARM ◄───── HEATING (ALM-02/ALM-03 직접)
      RESET (조건 충족)

  ※ ALM-04 (COMM_ERROR): 상태 전이 없음. 히터 즉시 OFF (FW 안전 모드)
  ※ ALM-05 (DOOR_OPEN): 상태 전이 없음. 히터 즉시 OFF (인라인 조건)
  ※ STOP 명령: HEATING / WARNING → IDLE
```

### 6.3 HEATING 동작 상세

- DS18B20 측정 주기: 750ms (12-bit 해상도)
- PID 출력: TIM3_CH3 Compare 값 (0 ~ 1000, 1kHz PWM)
- 팬: SP - 1°C 이상 → ON, SP - 3°C 미만 → OFF (과열 예방용 조건부 냉각)
- WARNING 타이머: 온도 > SP + warn_thr 순간 시작, 온도 복귀 시 초기화
- ALARM 타이머: 온도 > SP + alarm_thr 순간 시작

## 7. 알람 동작 상세

### 7.1 WARNING (ALM-01)

| 항목 | 내용 |
|-----|------|
| 판정 | 온도 > SP + warn_thr AND 타이머 ≥ warn_dur |
| FW 동작 | 부저 간헐 (0.5s ON / 0.5s OFF), PID 유지, EC에 `EVENT:WARN,ALM-01` 전송 |
| 자동 해제 | 온도 ≤ SP + warn_thr → 타이머 초기화, EC에 `EVENT:CLEAR,ALM-01` 전송 |
| 에스컬레이션 | WARNING 중 ALM-02 조건 충족 시 → ALARM 직접 진입 |

### 7.2 ALARM (ALM-02 / ALM-03)

| 항목 | 내용 |
|-----|------|
| 판정 (ALM-02) | 온도 > SP + alarm_thr AND 타이머 ≥ alarm_dur |
| 판정 (ALM-03) | DS18B20 읽기 실패 3회 연속 |
| FW 동작 | 히터 PWM=0%, 팬 ON, 부저 연속 ON, STATE = ALARM, EC에 `EVENT:ALARM,<alid>` 전송 |
| 복구 조건 | 온도 ≤ reset_thr (ALM-02) 또는 센서 정상 (ALM-03) |
| RESET 처리 | EC `RESET` 수신 → 복구 조건 확인 → 충족 시 IDLE 전이, 미충족 시 `NACK:RESET,<reason>` |

## 8. UART 통신 프로토콜

**물리 계층:** 115200 bps, 8N1, ST-LINK USB VCP
**프레임 형식:** ASCII 텍스트, `\r\n` 종단

### 8.1 FW → EC

| 메시지 | 형식 | 주기 | 설명 |
|--------|------|------|------|
| 데이터 | `DATA:<temp>,<sp>,<state>,<alarm>\r\n` | 1초 | 온도·SP·상태·알람 정기 보고 |
| Heartbeat | `HB\r\n` | 5초 | 통신 감시용 |
| 이벤트 | `EVENT:<type>,<alid>\r\n` | 발생 시 | WARN / ALARM / CLEAR |
| 명령 응답 | `ACK:<cmd>\r\n` | 수신 시 | 명령 수락 |
| 명령 거부 | `NACK:<cmd>,<reason>\r\n` | 수신 시 | 명령 거부 및 사유 |

**예시:**
```
DATA:28.3,30.0,HEATING,NONE\r\n
DATA:33.5,30.0,WARNING,ALM-01\r\n
DATA:35.8,30.0,ALARM,ALM-02\r\n
EVENT:WARN,ALM-01\r\n
EVENT:ALARM,ALM-02\r\n
EVENT:CLEAR,ALM-01\r\n
HB\r\n
ACK:START\r\n
NACK:RESET,TEMP_HIGH\r\n
```

### 8.2 EC → FW

| 명령 | 형식 | 설명 |
|------|------|------|
| 설정점 변경 | `SET:<sp>\r\n` | e.g., `SET:30.0\r\n` |
| 가열 시작 | `START\r\n` | IDLE → HEATING |
| 가열 정지 | `STOP\r\n` | HEATING/WARNING → IDLE |
| 복구 | `RESET\r\n` | ALARM → IDLE (조건 충족 시) |
| 상태 요청 | `STATUS\r\n` | 즉시 DATA 응답 요청 |

## 9. EC 알람 진단 패널

### 9.1 WARNING 패널 표시 항목

- 알람명: `ALM-01 TEMP_WARNING`
- 발생 시각 및 온도 스냅샷
- 현재 온도 / SP / warn_thr
- 조치 안내: "온도 추이를 모니터링하세요. ALARM 에스컬레이션에 주의."

### 9.2 ALARM 패널 표시 항목

- 알람명: `ALM-02 TEMP_ALARM` 또는 `ALM-03 SENSOR_ERROR`
- 발생 시각 및 온도 스냅샷
- 복구 조건 충족 여부 실시간 표시 (온도 실시간 비교)
- CS 점검 가이드 (TRG-001 참조)
- RESET 허용/거부 표시

### 9.3 EC 진단 화면 레이아웃 (ALARM 예시)

```
════════════════════════════════════════════════════════

════════════════════════════════════════════════════════

[Sensor] Temp: 35.8°C  SP: 30.0°C

────────────────────────────────────────────────────────

  3. 도어 닫힘 확인
```
