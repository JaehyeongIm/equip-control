## 3. 장비 개요

### 3.1 목적

본 장비는 반도체 공정 장비 구조를 모사한 **가상 공정 챔버**이다.  
Lock&Lock 밀폐용기를 챔버 바디로 사용하며, 히터 가열 → 과온 발생 → 인터락 → 복구의 전체 사이클을 실제 하드웨어로 구현한다.

주요 실습 대상:
- 과온(Over-Temperature) 장애 감지 및 인터락 동작
- 온도 상승 지연(No-Heat) 장애 진단 및 원인 분류
- 이중 인터락(HW + SW) 구조 동작 확인
- 센서 데이터 기반 현장 진단 절차

### 3.2 시스템 구성

| 계층 | 플랫폼 | 역할 |
|-----|--------|------|
| 1계층 (챔버 HW) | STM32 NUCLEO-F411RE + 주변 회로 | 센서 수집, 히터/팬/부저 제어, 인터락 실행 |
| 2계층 (EC) | macOS 터미널 프로세스 (C++) | GEM 상태 머신, 알람 판정, 인터락 명령, SECS/GEM |
| 3계층 (Host) | macOS 터미널 프로세스 (Python) | 운전 감시, 알람 표시, 명령 입력 (curses GUI) |

### 3.3 운전 범위

| 항목 | 범위 | 단위 |
|-----|------|------|
| 챔버 정상 운전 온도 | 25 ~ 60 | °C |
| 히터 설정 온도 | 30 ~ 60 | °C |
| 히터 전력 | 0 ~ 26 | W |
| PWM Duty | 0 ~ 100 | % |
| 히터 정격 전류 | ~1083 | mA |
| 운전 환경 온도 | 15 ~ 35 | °C |

## 5. 운전 모드 (Equipment State)

```
              ┌──────────────────────────────────┐
              │             IDLE                 │
              │  PWM=0%, 팬 OFF, 부저 OFF         │
              │  LED: 소등                        │
              └────────────┬─────────────────────┘
                           │ START 명령
                           ▼
              ┌──────────────────────────────────┐
              │           HEATING                │
              │  PWM 0~100%, 설정 온도 추종       │
              │  LED: 점등                        │
              └────┬───────────────────┬─────────┘
                   │ 온도 > 65°C        │ STOP 명령
                   ▼                   ▼
              ┌─────────┐         ┌────────┐
              │  ALARM  │         │  IDLE  │
              │ PWM=0%  │         └────────┘
              │ 팬 ON   │
              │ 부저 ON │
              └────┬────┘
                   │ 온도 > 68°C 또는 도어 열림
                   ▼
              ┌──────────────────────────────────┐
              │          INTERLOCK               │
              │  PWM=0%, 팬 ON, 부저 ON           │
              │  LED: 고속 점멸                   │
              │  SECS/GEM S5F1 알람 보고          │
              └────────────┬─────────────────────┘
                           │ 복구 조건 충족 + RESET 명령
                           ▼
                        IDLE
```

| 모드 | PWM | 팬 | 부저 | LED | 진입 조건 |
|-----|-----|-----|------|-----|---------|
| IDLE | 0% | OFF | OFF | 소등 | 초기 상태, STOP 명령, 복구 완료 |
| HEATING | 0~100% | OFF (정상 시) | OFF | 점등 | START 명령 (IDLE에서) |
| ALARM | 0% | ON | ON | 저속 점멸 | 온도 > 65°C, NO_HEAT 감지 |
| INTERLOCK | 0% | ON | ON | 고속 점멸 | 온도 > 68°C, 도어 열림 |

## 6. 기능 블록 정의

### 6.1 히터 가열 제어 (PWM)

- **목적**: 챔버 온도를 설정점까지 가열하고 유지
- **방식**: MOSFET PWM (1kHz, TIM3_CH3, PB0)
- **제어 알고리즘**: Bang-bang 또는 단순 비례 (설정 온도 미만 시 PWM_MAX, 초과 시 PWM 감소)
- **정상 조건**: HEATING 모드, 도어 닫힘, 온도 < 65°C

### 6.2 온도 감시 (SHT31)

- **목적**: 챔버 온도 실시간 측정 및 임계값 비교
- **주기**: 1초 (FW TaskSensor)
- **EC 판정**: 매 샘플마다 SP-01(65°C), SP-02(68°C) 비교
- **연속 오류 감지**: I2C 읽기 3회 연속 실패 → SENSOR_ERROR 알람

### 6.3 전류 감시 (INA219)

- **목적**: 히터 회로 전류 측정으로 ST-22 동작 및 단선 감지
- **주기**: 1초 (센서 데이터에 포함)
- **판정 기준**:
  - HEATING 중 전류 < SP-05(50mA) → HW_INTERLOCK 알람 (ST-22 동작 또는 단선)
  - HEATING 중 전류 < 250mA (정격 30%) → CURRENT_ANOMALY 알람

### 6.4 냉각 팬

- **목적**: ALARM/INTERLOCK 발생 시 챔버 강제 냉각
- **동작**: ALARM 또는 INTERLOCK 진입 시 자동 ON
- **복구**: IDLE 전이 시 자동 OFF

### 6.5 부저 알람

- **목적**: ALARM/INTERLOCK 발생을 청각적으로 운영자에게 알림
- **동작**: ALARM 또는 INTERLOCK 진입 시 자동 ON
- **복구**: IDLE 전이 시 자동 OFF

### 6.6 상태 LED (LD2, PA5)

| 모드 | LED 동작 |
|-----|---------|
| IDLE | 소등 |
| HEATING | 점등 (상시) |
| ALARM | 저속 점멸 (500ms 주기) |
| INTERLOCK | 고속 점멸 (200ms 주기) |

### 6.7 Heartbeat 감시

- STM32 TaskHeartbeat가 5초 주기로 MSG_HEARTBEAT_REQ 전송
- EC에서 10초(SP-10) 이상 미수신 시 통신 단선 판정 → COMM_ERROR 알람

## 7. 메인 시나리오: 과온 (Over-Temperature)

### 7.1 시나리오 설명

챔버 밀폐 상태에서 히터를 장시간 가동하거나 PWM Duty를 높게 설정하여 온도를 상승시킨다.  
65°C → ALARM, 68°C → SW INTERLOCK, 70°C → ST-22 HW 차단의 순서로 동작을 검증한다.

### 7.2 기능 흐름

```
HEATING 모드 운전 중
       │
       ▼
SHT31 온도 > 65°C (SP-01)
       │
       ├── EC 판정: ALARM 전이
       ├── PWM Duty = 0% (즉시)
       ├── 팬 ON, 부저 ON
       ├── ALID=1 (TEMP_HIGH) S5F1 보고
       │
       ▼ (온도 계속 상승 시)
SHT31 온도 > 68°C (SP-02)
       │
       ├── EC 판정: INTERLOCK 전이
       ├── ALID=2 (TEMP_CRITICAL) S5F1 보고
       │
       ▼ (온도 계속 상승 시)
챔버 70°C 도달 → ST-22 개방 (HW 차단)
       │
       ├── INA219 전류 < 50mA 감지
       ├── ALID=4 (HW_INTERLOCK) 알람
       └── EC 화면: "HW 인터락 동작 — ST-22 확인"
```

### 7.3 복구 흐름

```
냉각 팬 동작 → 챔버 온도 하강
       │
       ▼
온도 < 50°C (SP-04) + INA219 전류 정상 복귀 + 도어 닫힘
       │
       ▼
Host: RESET 명령 입력
       │
       ▼
EC: 복구 조건 확인 → 모두 충족 시 IDLE 전이
       │
       ▼
장비 정상 운전 재개
```

## 8. Host 진단 패널 기능 명세

### 9.1 개요

알람(S5F1) 수신 시 Host는 단순 알람 표시에 그치지 않고 **자동 진단 패널을 활성화**한다.  
진단 패널은 알람 수신 시점의 센서 스냅샷을 캡처하여 원인 후보를 자동 분류하고,  
TRG-001 기반의 현장 점검 체크리스트와 복구 조건 충족 여부를 실시간으로 표시한다.

**목적:** CS 엔지니어가 Host 화면 하나로 원인 특정 → 조치 → 복구 검증까지  
단계별로 가이드받을 수 있도록 하여 현장 대응 시간을 단축한다.

---

### 9.2 진단 데이터 수집

S5F1(AlarmSet=True) 수신 시점에 Host는 다음 데이터를 **스냅샷으로 캡처**한다.

| 수집 항목 | 소스 | 비고 |
|---------|------|------|
| ALID | S5F1 메시지 | 알람 식별자 |
| SHT31 온도 (T_alarm) | 직전 S6F11 CEID-8 수신값 | 알람 발생 시점 온도 |
| INA219 전류 (I_alarm) | 직전 S6F11 CEID-8 수신값 | 알람 발생 시점 전류 |
| 알람 발생 시각 | Host 시스템 시각 | |

---

### 9.3 ALID별 원인 후보 자동 분류

스냅샷의 `I_alarm` 값을 기준으로 원인 후보를 순위별로 자동 표시한다.  
분류 기준은 AIM-001 §5, TRG-001 §4.2·§5.2를 따른다.

#### ALID=1 (TEMP_HIGH, 65°C 초과)

| I_alarm 범위 | 1순위 원인 | 2순위 원인 |
|------------|---------|---------|
| ≥ 500mA | PWM Duty 과다 / 챔버 밀폐 과다 | 냉각 팬 미동작 |
| < 50mA | ST-22 조기 동작 | INA219 측정 오류 |

#### ALID=2 (TEMP_CRITICAL, 68°C 초과)

| I_alarm 범위 | 1순위 원인 | 2순위 원인 |
|------------|---------|---------|
| ≥ 500mA | 히터 출력 과잉, 강제 냉각 필요 | 챔버 밀폐 불량 |
| < 50mA | ST-22 동작 (70°C 이전 조기 동작 가능성) | 배선 불량 |

#### ALID=4 (HW_INTERLOCK, INA219 전류 소실)

| 조건 | 원인 |
|-----|------|
| T_alarm ≥ 70°C | ST-22 정상 동작 (바이메탈 개방) |
| T_alarm < 70°C | ST-22 이상 동작 또는 히터 배선 탈거 |

---

### 9.4 현장 점검 체크리스트

원인 후보 분류 후 ALID별 점검 항목을 체크리스트 형식으로 표시한다.  
운영자는 `CHECK <번호>` 명령으로 각 항목을 완료 처리한다.

#### ALID=1, 2 공통 점검 항목

```
[ ] 1. 챔버 도어 닫힘 확인
[ ] 2. 냉각 팬 동작 확인 (회전 소리/진동)
[ ] 3. Host 화면 INA219 전류값 확인
[ ] 4. 챔버 주변 환경 온도 확인
```

#### ALID=4 점검 항목

```
[ ] 1. ST-22 양단 도통 확인 (냉각 후 자동 복귀 대기)
[ ] 2. INA219 전류 복귀 확인 (> 800mA)
[ ] 3. 히터 배선 연결 상태 육안 확인
```

---

### 9.5 복구 조건 추적 및 RESET 허용/거부

Host는 RESET 명령 입력 시 다음 조건을 자동 확인하여 결과를 표시한다.

| 조건 | 확인 방법 | 미충족 시 메시지 |
|-----|---------|-------------|
| 온도 < 50°C (SP-04) | 최신 S6F11 수신 온도값 | "온도 미충족 (현재 XX.X°C, 기준 50°C)" |
| 체크리스트 전항목 완료 | `CHECK` 명령 완료 여부 | "미완료 점검 항목: N건" |

- **조건 전체 충족 시**: `RESET` 명령을 EC로 전송 (S2F41 RESET)
- **조건 미충족 시**: EC로 전송하지 않고 Host 화면에 미충족 사유 표시
