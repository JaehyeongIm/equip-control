## 3. 인터락 분류 정의

| 분류 | 기호 | 설명 |
|-----|------|------|
| 하드웨어 인터락 | HW-IL | 소프트웨어와 무관하게 물리적으로 회로를 차단. ST-22 바이메탈 서모스탯이 직접 개방. |
| 소프트웨어 인터락 | SW-IL | EC 소프트웨어가 INTERLOCK 상태로 전이하여 PWM Duty=0% 명령 전송. RESET 명령으로만 해제. |
| 소프트웨어 알람 | SW-AL | EC 소프트웨어가 ALARM 상태로 전이. 일부 조건 해소 시 자동 해제 또는 RESET으로 해제. |

## 4. 알람 / 인터락 매트릭스

### 4.1 전체 알람 목록

| ALID | 알람명 | 발생 조건 | 감지 소스 | 인터락 분류 | 자동 대응 | 관련 시나리오 |
|------|--------|---------|---------|-----------|---------|------------|
| 1 | TEMP_HIGH | SHT31 온도 > 65°C | SHT31 (EC 판정) | SW-AL | PWM=0%, 팬 ON, 부저 ON, S5F1 보고 | 과온 |
| 2 | TEMP_CRITICAL | SHT31 온도 > 68°C | SHT31 (EC 판정) | SW-IL | PWM=0%, S5F1 보고, INTERLOCK 전이 | 과온 |
| 4 | HW_INTERLOCK | INA219 전류 < 50mA (PWM ON 중) | INA219 (EC 판정) | HW-IL + SW-AL | EC 화면 경고, 운영자 확인 요구 | 과온 (ST-22 동작) |
| 5 | SENSOR_ERROR | SHT31 I2C 오류 연속 3회 | SHT31 I2C | SW-IL | PWM=0%, S5F1 보고 | 공통 |
| 6 | COMM_ERROR | Heartbeat 10초 미수신 | UART (EC 판정) | SW-AL | 제어 명령 차단, S5F1 보고 | 공통 |

> ALID 3(DOOR_OPEN), 7(PM_DUE), 8(NO_HEAT)은 본 프로젝트 범위에서 구현하지 않는다.

## 5. 시나리오별 알람 발생 순서

### 5.1 메인 시나리오: 과온 (Over-Temperature)

```
정상 HEATING 운전
        │
        │ 온도 65°C 초과
        ▼
[ALID=1 TEMP_HIGH] ─── SW-AL
  PWM=0%, 팬 ON, 부저 ON
  EC → ALARM 상태 전이
  S5F1 (ALID=1, AlarmSet=True) 전송
        │
        │ 온도 68°C 초과 (계속 상승)
        ▼
[ALID=2 TEMP_CRITICAL] ─── SW-IL
  EC → INTERLOCK 상태 전이
  S5F1 (ALID=2, AlarmSet=True) 전송
        │
        │ 온도 70°C 도달 (HW 동작)
        ▼
ST-22 개방 (물리적 히터 회로 차단)
        │
        ▼
INA219 전류 < 50mA 감지
        │
        ▼
[ALID=4 HW_INTERLOCK] ─── HW-IL 확인
  EC 화면: "HW 인터락 동작 — ST-22 확인 필요"
  운영자 직접 확인 및 복구 필요
```

## 6. 이중 인터락 계층도

```
온도 레벨별 대응 계층

65°C ────────── SW ALARM (ALID=1)
                EC 소프트웨어 감지, PWM=0%
                │
68°C ────────── SW INTERLOCK (ALID=2)
                EC 소프트웨어 감지, INTERLOCK 상태
                RESET 명령으로만 복구
                │
70°C ────────── HW INTERLOCK (ST-22 동작)
                물리적 회로 개방, 소프트웨어와 독립 동작
                온도 하강 시 자동 복귀 (접점 재연결)
                INA219로 동작 여부 확인 가능 (ALID=4)
```

**이중 인터락의 목적:**
- SW-IL 단독: SW 버그 또는 EC 프로세스 이상 시 히터 제어 불능 위험
- HW-IL 단독: SW 감지 없이 HW만 차단되면 진단 정보 없음
- **두 계층 병존**: SW가 먼저 차단(65°C, 68°C)하여 HW 동작(70°C) 방지, HW는 최후 안전망

## 7. 복구 조건 매트릭스

| ALID | 알람명 | 복구 조건 | 복구 명령 | 비고 |
|------|--------|---------|---------|------|
| 1 | TEMP_HIGH | 온도 < 50°C (SP-04) | RESET | 자동 해제 불가, 운영자 RESET 필요 |
| 2 | TEMP_CRITICAL | 온도 < 50°C + INA219 전류 정상 | RESET | INTERLOCK → IDLE 전이 |
| 4 | HW_INTERLOCK | ST-22 복귀 확인 (INA219 전류 정상) + 온도 < 50°C | RESET | HW 직접 확인 후 RESET |
| 5 | SENSOR_ERROR | SHT31 연속 3회 정상 읽기 확인 | RESET | I2C 배선 점검 후 RESET |
| 6 | COMM_ERROR | Heartbeat 재수신 확인 | 자동 복구 | UART 케이블 재연결 |
**공통 복구 조건 (모든 RESET에 적용):**
1. 온도 < 50°C (SP-04)
2. 도어 닫힘 (PA1 HIGH)
3. 해당 알람의 개별 복구 조건 (위 테이블 참조)

> 복구 조건 미충족 시 RESET 명령을 거부하고 미충족 항목을 Host 화면에 표시한다.
