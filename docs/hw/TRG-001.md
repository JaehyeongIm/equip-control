## 1. 적용 범위

본 문서는 가상 공정 챔버 장비에서 발생하는 주요 장애에 대한 현장 대응 절차를 정의한다.  
CS 엔지니어가 알람 발생부터 원인 특정, 복구 조치, 재가동 검증까지 수행하기 위한 단계별 절차서이다.

**대상 시나리오:**
- 과온 (Over-Temperature) — ALID 1, 2, 4

## 3. 안전 주의사항

> **[경고]** 장애 대응 전 반드시 아래 사항을 확인할 것.

1. **전원 차단 확인**: 히터 회로(24V) 작업 전 파워 서플라이 출력 OFF
2. **챔버 온도 확인**: 챔버 온도 50°C 미만 확인 후 접근
3. **ST-22 상태**: 바이메탈 재설정(자동복귀) 전 챔버가 충분히 냉각됐는지 확인
4. **MOSFET 방열**: MOSFET(IRLZ44NPBF)이 발열 상태일 수 있음. 터치 주의
5. **브레드보드 배선**: 작업 시 24V 레일과 3.3V/5V 레일 교차 접촉 방지

## 4. 메인 시나리오: 과온 장애 대응

### 4.1 알람 확인

| 항목 | 내용 |
|-----|------|
| 관련 ALID | 1 (TEMP_HIGH), 2 (TEMP_CRITICAL), 4 (HW_INTERLOCK) |
| 초기 증상 | Host 화면 ALARM/INTERLOCK 전이, 부저 경보, 팬 동작 |
| 장비 상태 | PWM=0% (히터 OFF), 팬 ON, 부저 ON |

**Host 화면 확인 항목:**
```
[State]  Equipment: INTERLOCK   | Online: YES
[Sensor] Temp: XX.X°C  Curr: X mA
[Alarm]  Active: 2  (ALID 1, 2)
[GUIDE]  챔버 냉각 대기 — 온도 50°C 미만 확인 후 RESET
```

---

### 4.2 원인 분류 진단 트리

```
ALID 1 또는 2 (과온) 알람 발생
         │
         ▼
[Step 1] SHT31 온도 값 확인 (Host 화면)
         │
         ├── 온도 정상 (< 65°C) ──► SHT31 센서 오류 또는 EC 소프트웨어 이상
         │                           → TSG-03 (센서 오류) 참조
         │
         └── 온도 실제 상승 ──────► [Step 2]
         │
         ▼
[Step 2] INA219 전류 값 확인
         │
         ├── 전류 ≥ 500mA ──────► 히터 정상 동작 중 과온
         │                         원인: PWM Duty 과다 / 챔버 밀폐 / 냉각 불량
         │                         → [Step 3-A]
         │
         └── 전류 < 50mA ────────► ST-22 동작 또는 회로 차단 (ALID=4)
                                    원인: 챔버 70°C 도달 → HW 인터락 동작
                                    → [Step 3-B]
```

#### Step 3-A: 히터 동작 중 과온

```
원인 가능성:
  1. PWM Duty 설정 과다 → 히터 출력 과잉
  2. 챔버 도어 열림 → 냉각 효율 저하 / 센서 위치 이탈
  3. 냉각 팬 미동작 → 방열 불량
  4. 환경 온도 상승 → 챔버 주변 고온

점검 순서:
  ① Host 화면에서 PWM Duty 설정값 확인
  ② 도어 닫힘 상태 육안 확인
  ③ 냉각 팬 동작 확인 (팬 날개 회전 소리/진동)
  ④ 장비 주변 환경 온도 확인

조치:
  - PWM Duty 감소 (설정점 온도 하향)
  - 팬 미동작 시 → TSG-FAN 참조 (IRF520 구동 불량)
  - 도어 밀폐 확인 및 닫기
```

#### Step 3-B: ST-22 동작 (ALID=4)

```
INA219 전류 < 50mA, PWM ON 상태

원인: 챔버 온도가 70°C에 도달하여 ST-22 바이메탈 개방

점검:
  ① 챔버 온도 > 70°C 여부 확인 (이미 인터락 상태이므로 냉각 중일 것)
  ② 챔버 표면 온도 접촉 측정 (선택)

처치:
  ① 팬 동작 유지 (자동 ON 상태)
  ② 온도 < 50°C까지 냉각 대기
  ③ ST-22는 자동복귀형 — 온도 하강 시 접점 자동 복귀
     (INA219 전류 복귀 확인으로 ST-22 복귀 판정)
```

---

### 4.3 복구 절차 (메인 시나리오)

```
Step 1. 냉각 대기
  ──────────────────────────────────────────────────
  팬 자동 동작 확인 → Host 화면 온도 모니터링
  온도 < 50°C 도달까지 대기 (SP-04)

Step 2. ST-22 복귀 확인 (ALID=4가 발생한 경우)
  ──────────────────────────────────────────────────
  Host 화면: INA219 Current 값 확인
  ALARM 상태에서 전류 > 800mA로 복귀 시 ST-22 접점 재연결 확인됨
  전류 복귀 없을 시: 히터 배선 또는 ST-22 자체 불량 가능성 → 배선 점검

Step 3. 복구 조건 최종 확인
  ──────────────────────────────────────────────────
  ✓ 온도 < 50°C
  ✓ INA219 전류 정상 범위 (800mA ~ 1300mA) 또는 히터 OFF 상태에서 0mA
  ✓ 도어 닫힘 (PA1 정상)

Step 4. RESET 명령 입력
  ──────────────────────────────────────────────────
  Host CLI: > RESET
  EC 복구 조건 확인 → 전체 충족 시 IDLE 전이
  조건 미충족 시: 거부 사유 표시 → 해당 항목 추가 점검

Step 5. 정상 운전 재개 확인
  ──────────────────────────────────────────────────
  Host 화면: State = IDLE
  START 명령 → HEATING 전이 확인
  온도 정상 상승 및 INA219 전류 정상 확인
```

## 5. 일반 장애 — 통신 단선 (ALID=6)

### 5.1 현상 및 원인

| 현상 | 원인 |
|------|------|
| Host 화면 Online: NO | EC ↔ STM32 UART 단선 |
| ALID=6 COMM_ERROR 발생 | Heartbeat 10초 미수신 |
| START/STOP 명령 거부 | 통신 두절 상태에서 장비 제어 금지 |

### 5.2 조치 절차

```
① USB 케이블 연결 상태 확인
  → STM32 NUCLEO ST-LINK USB 케이블 체결 확인

② macOS에서 장치 인식 확인
  → ls /dev/tty.usbmodem* 명령으로 포트 확인

③ EC 재시작 (포트 인식 안 될 경우)
  → Ctrl+C 후 ./build/ec /dev/tty.usbmodem* 재실행

④ 통신 복구 확인
  → Heartbeat 수신 재개 → COMM_ERROR 자동 해제
  → Host 화면 Online: YES 복귀
```
