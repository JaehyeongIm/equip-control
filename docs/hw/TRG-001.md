## 1. 적용 범위

본 문서는 소형 챔버 온도제어 및 알람진단 시스템에서 발생하는 주요 장애에 대한 현장 대응 절차를 정의한다.

CS 엔지니어가 알람 발생부터 원인 특정, 복구 조치, 재가동 검증까지 수행하기 위한 단계별 절차서이다.

**대상 알람:**
- ALM-01: TEMP_WARNING (과온 경고)
- ALM-02: TEMP_ALARM (과온 알람, 히터 정지)
- ALM-03: SENSOR_ERROR (DS18B20 오류)
- ALM-04: COMM_ERROR (EC ↔ FW 통신 단선)
- ALM-05: DOOR_OPEN (도어 열림 / 배선 단선)

## 3. 안전 주의사항

> **[경고]** 장애 대응 전 반드시 아래 사항을 확인할 것.

1. **전원 차단 확인**: 히터 회로(24V) 작업 전 파워 서플라이 출력 OFF
2. **챔버 온도 확인**: ALARM 발생 후 챔버가 충분히 냉각(RESET 허용 온도 이하)됐는지 확인 후 접근
3. **MOSFET 발열**: MOSFET 모듈이 발열 상태일 수 있음. 터치 주의
4. **브레드보드 배선**: 24V 레일과 3.3V 레일 교차 접촉 방지
5. **PSU 전류 제한**: 단락 보호를 위해 1.5A 제한 유지

## 4. 메인 시나리오: 과온 (ALM-01 / ALM-02)

### 4.1 알람 확인

| 항목 | TEMP_WARNING (ALM-01) | TEMP_ALARM (ALM-02) |
|-----|----------------------|---------------------|
| EC 표시 | STATE: WARNING, 부저 간헐 | STATE: ALARM, 부저 연속 |
| 히터 | PID 유지 | OFF (PWM=0%) |
| 팬 | 조건부 ON | ON |
| 복구 | 자동 (온도 복귀 시) | EC RESET 수동 |

**EC 화면 예시 (TEMP_ALARM):**
```
[State]  FW State: ALARM       | UART: Connected
[Sensor] Temp: 35.8°C  SP: 30.0°C
[Alarm]  ALM-02 TEMP_ALARM  |  발생: 14:23:11
[복구 조건]  온도 ≤ 28.0°C : ✗ (현재 35.8°C)
```

---

### 4.2 원인 분류 진단 트리

```
TEMP_WARNING / TEMP_ALARM 발생
         │
         ▼
[Step 1] 온도 값이 실제로 상승했는가?
         │
         ├── 아니오 (온도 정상) ──► DS18B20 오류 가능성
         │                         → 4절 SENSOR_ERROR 참조
         │
         └── 예 (온도 실제 상승) ──► [Step 2]
         │
         ▼
[Step 2] 히터 과출력 확인
         │
         ├── SP 대비 현저한 온도 초과 ──► PWM Duty 과다 또는 챔버 밀폐 과다
         │                               → [Step 3-A]
         │
         └── 완만한 온도 상승 ──────────► 환경 온도 상승 또는 팬 미동작
                                          → [Step 3-B]
```

#### Step 3-A: 히터 과출력 과온

```
원인 가능성:
  1. PID Kp 과대 → 히터 출력 과잉 → 오버슈트 큰 경우
  2. 챔버 밀폐 상태에서 열 축적
  3. 팬이 동작하지 않아 냉각 불량

점검 순서:
  ① EC 화면: 현재 온도와 SP 값 확인
  ② 팬 동작 여부 확인 (소리/진동)
  ③ 챔버 밀폐 상태 확인 (도어 닫힘)

조치:
  - PID 파라미터 재조정 (Kp 감소 또는 SP 하향)
  - 팬 미동작 시 → 5절 팬 미동작 참조
```

#### Step 3-B: 환경 온도 / 팬 문제

```
원인 가능성:
  1. 실내 온도 상승 → 챔버 주변 열 축적
  2. 팬 동작 부족 → 강제 냉각 불충분

점검 순서:
  ① 장비 주변 환경 온도 확인
  ② 팬 동작 확인

조치:
  - 장비 주변 환기 개선
  - SP 재설정 (현 환경 온도 감안)
```

---

### 4.3 복구 절차 (TEMP_ALARM)

```
Step 1. 냉각 대기
  ──────────────────────────────────────────────────
  팬 자동 ON 확인 → EC 화면 온도 모니터링
  온도 ≤ reset_thr (기본 SP - 2°C) 도달까지 대기

Step 2. 복구 조건 최종 확인
  ──────────────────────────────────────────────────
  ✓ 온도 ≤ reset_thr
  ✓ 도어 닫힘 (PA1 LOW)

Step 3. RESET 명령 입력
  ──────────────────────────────────────────────────
  EC CLI: > RESET
  조건 충족 시: ACK:RESET → STATE = IDLE
  조건 미충족 시: NACK:RESET,TEMP_HIGH → 추가 냉각 대기

Step 4. 정상 운전 재개 확인
  ──────────────────────────────────────────────────
  EC: STATE = IDLE
  EC CLI: > START → STATE = HEATING 전이 확인
  온도 정상 상승 및 SP 추종 확인
```

## 5. SENSOR_ERROR (ALM-03)

### 5.1 현상 및 원인

| 현상 | 원인 |
|------|------|
| EC STATE: ALARM, ALM-03 | DS18B20 배선 단선 또는 불량 |
| EC: `EVENT:ALARM,ALM-03` | OneWire CRC 오류 3회 연속 |
| 히터 자동 OFF | 온도 미측정 상태에서 히터 가동 불가 (안전) |

### 5.2 조치 절차

```
① EC 화면에서 ALM-03 SENSOR_ERROR 확인

② DS18B20 배선 점검
  → PB5 (D4) - DS18B20 DQ 연결 확인
  → 4.7kΩ 풀업 저항 연결 확인 (DQ ↔ 3.3V)
  → DS18B20 VDD, GND 연결 확인

③ 브레드보드 접촉 확인
  → 점퍼선이 브레드보드 홀에 제대로 삽입됐는지 확인

④ 배선 복구 후 EC RESET
  → EC CLI: > RESET
  → ACK:RESET 수신 시 STATE = IDLE 복귀
  → DS18B20 온도 DATA 정상 수신 확인

⑤ SENSOR_ERROR 재발 시
  → DS18B20 소자 교체 (정품 여부 확인)
  → 배선 길이 최소화 (OneWire는 긴 배선에서 노이즈 취약)
```

## 6. COMM_ERROR (ALM-04)

### 6.1 현상 및 원인

| 현상 | 원인 |
|------|------|
| EC: UART 연결 끊김 표시 | ST-LINK USB 케이블 탈거 |
| FW 히터 자동 OFF | EC Heartbeat 10초 미수신 → FW 안전 모드 |
| EC 재시작 필요 | EC 프로세스 크래시 |

### 6.2 조치 절차

```
① USB 케이블 연결 상태 확인
  → STM32 NUCLEO ST-LINK USB 케이블 체결 확인

② macOS에서 장치 인식 확인
  → ls /dev/tty.usbmodem* 명령으로 포트 확인

③ EC 재시작 (포트 인식 안 될 경우)
  → Ctrl+C 후 ./ec /dev/tty.usbmodem* 재실행

④ 통신 복구 확인
  → EC: UART Connected 표시
  → FW: HB 수신 재개 → 안전 모드 자동 해제
  → EC: START 명령 후 HEATING 전이 확인
```

## 7. DOOR_OPEN (ALM-05)

### 7.1 현상 및 원인

| 현상 | 원인 |
|------|------|
| EC: `EVENT:WARN,ALM-05` 수신 | 도어 스위치 열림 |
| 히터 자동 OFF | PA1 = HIGH (도어 열림 또는 배선 단선) |

### 7.2 조치 절차

```
① 도어 스위치 물리 상태 확인
  → 스위치 눌림 여부 육안 확인

② 배선 확인
  → PA1 (A1) ↔ 스위치 NO 단자 연결 확인
  → 스위치 COM ↔ GND 연결 확인

③ 도어 닫힘 확인 후 자동 해제
  → 스위치 눌림 → PA1 = LOW → ALM-05 자동 해제
  → EC: `EVENT:CLEAR,ALM-05` 수신 확인
  → 히터 PID 자동 재개
```

## 8. 일반 진단 — 팬 미동작

### 8.1 현상 및 점검

```
현상: ALARM 상태에서 팬이 회전하지 않음

점검 순서:
  ① PA0 (A0) GPIO 출력 HIGH 확인
     → EC START 후 멀티미터로 PA0 핀 전압 측정 (HIGH = 3.3V)

  ② 릴레이 모듈 동작 확인
     → IN 단자 전압 확인 (3.3V)
     → 딸깍 소리 확인 (릴레이 코일 동작)
     → 릴레이 출력 단자 전압 확인 (NO - COM 도통)

  ③ DC-DC 컨버터 출력 확인
     → OUT+ 단자 전압 = 5V 확인
     → 입력 전압 = 24V 확인 (5V 입력으로는 5V 출력 불가)

  ④ 팬 자체 확인
     → 5V 직접 인가 후 회전 여부 확인
```
