# 1. 테스트 전략

## 1.1 테스트 수준

| 수준 | 대상 | 도구 / 방법 |
|------|------|------------|
| 단위 (Unit) | STM32 모듈 함수 (CRC, 프레임 파서, 상태 머신) | STM32CubeIDE, 시리얼 모니터 |
| 통합 (Integration) | STM32↔EC UART, EC↔Host HSMS | Wireshark, 시리얼 로거 |
| 시스템 (System) | 3계층 전체 엔드투엔드 시나리오 | Host CLI + EC 로그 + STM32 로그 |
| 수용 (Acceptance) | AC-001 ~ AC-016 정량 기준 | 수동 검증 + 타임스탬프 로그 분석 |

## 1.2 테스트 우선순위

| 등급 | 항목 | 이유 |
|------|------|------|
| P1 (필수) | 통신 신뢰성, 상태 머신, Watchdog | 시스템 무결성 직결 |
| P2 (중요) | 알람/이벤트, 인터록, Graceful Degradation | 운영 안전성 직결 |
| P3 (일반) | Host 표시, 조회 응답, 디버그 로그 | 사용성 관련 |

## 1.3 결과 판정 기준

- **PASS**: 수용 기준(AC) 전항목 충족
- **FAIL**: 수용 기준 미충족 (정량 기준 위반 포함)
- **SKIP**: 하드웨어 미연결 등 환경 제약으로 실행 불가

# 2. 테스트 환경

## 2.1 하드웨어 구성

```
STM32 NUCLEO-F411RE
  ├── SHT31 (온습도 센서) — I2C
  ├── INA219 (전류/전압 센서) — I2C
  ├── 팬 제어 GPIO (PA5)
  ├── 부저 GPIO
  ├── LED GPIO
  └── 버튼 (PC13)
      │
      │ USB Virtual COM (UART 115200)
      ▼
Windows PC (EC 소프트웨어)
      │
      │ HSMS TCP/IP (기본 포트: 5000)
      ▼
macOS PC (Host 프로그램)
```

## 2.2 소프트웨어 요구사항

| 항목 | 사양 |
|------|------|
| STM32 펌웨어 빌드 | STM32CubeIDE (Release 빌드) |
| EC 소프트웨어 OS | Windows 10/11 |
| Host 프로그램 OS | macOS 12 이상 |
| 시리얼 모니터 | Tera Term 또는 minicom |
| 패킷 캡처 | Wireshark (HSMS TCP 분석) |

## 2.3 사전 조건

- [ ] STM32 펌웨어가 NUCLEO-F411RE에 플래시됨
- [ ] 모든 센서 및 액추에이터 배선 확인 완료
- [ ] EC 소프트웨어 실행 전 COM 포트 번호 설정 확인
- [ ] Host PC에서 EC IP 주소 및 포트 설정 확인
- [ ] 두 PC가 동일 네트워크에 연결됨

## TC-FW-001 센서 데이터 수집 및 전송

| 항목 | 내용 |
|------|------|
| **추적** | FR-001, FR-002, FR-003, FR-009 / NFR-003, NFR-009 / AC-006 |
| **우선순위** | P1 |
| **분류** | 통합 |

**목적**: SHT31 및 INA219 센서 데이터가 1초 주기로 수집되어 EC에 전송되는지 검증

**사전 조건**: STM32 전원 인가, 센서 정상 연결, EC 소프트웨어 실행

**절차**:
1. STM32 시리얼 로그를 모니터링한다.
2. EC 수신 로그에서 SENSOR_DATA(Type=0x01) 메시지를 관찰한다.
3. 수신 타임스탬프 간격을 10분 이상 수집한다.
4. 온도, 습도, 전류, 전압 값이 정상 범위에 있는지 확인한다.

**합격 기준**:
- 센서 데이터 수신 주기: **1,000ms ± 100ms** (NFR-003)
- TaskSensor 수집 jitter: **±50ms 이내** (NFR-009)
- 온도 범위: −40°C ~ 125°C (FR-004)
- 습도 범위: 0% ~ 100% (FR-004)
- 전류 범위: 0mA ~ 3,200mA (FR-004)
- 전압 범위: 0V ~ 26V (FR-004)

**확인 사항**:
- [ ] 주기 1,000ms ± 100ms 충족
- [ ] jitter ±50ms 이내 (로그 분석)
- [ ] 센서 값 정상 범위

## TC-FW-002 센서 유효성 검사 및 오류 보고

| 항목 | 내용 |
|------|------|
| **추적** | FR-004, FR-025 / NFR-007, NFR-011 / AC-015 |
| **우선순위** | P1 |
| **분류** | 통합 |

**목적**: 센서 범위 이탈 또는 I2C 실패 시 오류 코드가 포함된 메시지가 전송되는지 검증

**절차**:
1. SHT31 센서 I2C 배선을 의도적으로 분리한다.
2. EC 수신 로그에서 오류 코드(sht31_error ≠ 0) 메시지를 확인한다.
3. INA219 데이터가 계속 수신되는지 확인한다. *(Graceful Degradation)*
4. 배선을 복원하고 오류 코드가 사라지는지 확인한다.

**합격 기준**:
- SHT31 장애 시 `sht31_error` 필드에 오류 코드 포함
- INA219 수집 중단 없음 (NFR-011)
- 복원 후 다음 수집 주기부터 정상 데이터 수신

**확인 사항**:
- [ ] 오류 코드 정상 전송
- [ ] INA219 동작 유지
- [ ] 복원 후 자동 정상화

## TC-FW-003 액추에이터 제어 (팬 / 부저 / LED)

| 항목 | 내용 |
|------|------|
| **추적** | FR-005, FR-006, FR-007 / NFR-009 / AC-004 |
| **우선순위** | P1 |
| **분류** | 통합 |

**목적**: EC 명령에 따라 팬, 부저, LED가 정상 동작하는지 검증

**절차**:
1. EC에서 CMD_FAN(0x10, ON) 전송 후 팬 동작 확인
2. EC에서 CMD_FAN(0x10, OFF) 전송 후 팬 정지 확인
3. EC에서 CMD_BUZZER(0x11, ON) 전송 후 부저 동작 확인
4. EC에서 CMD_LED(0x12) 각 상태 전송 후 LED 상태 확인:
   - 0x01 (IDLE), 0x02 (RUN), 0x03 (ALARM), 0x04 (ERROR)
5. 명령 수신 후 ACK 반환 시간 측정

**합격 기준**:
- 팬 ON/OFF 즉각 반응
- 부저 ON/OFF 즉각 반응
- LED 4가지 상태 구분 표시
- 명령 실행: 수신 후 **50ms 이내** (FR-011)
- ACK 반환: 수신 후 **100ms 이내** (FR-011)

**확인 사항**:
- [ ] 팬 제어 동작
- [ ] 부저 제어 동작
- [ ] LED 4상태 표시
- [ ] ACK 100ms 이내

## TC-FW-005 UART 통신 프로토콜 신뢰성

| 항목 | 내용 |
|------|------|
| **추적** | FR-012, FR-013, FR-014, FR-015, FR-016 / NFR-004 / AC-012 |
| **우선순위** | P1 |
| **분류** | 통합 |

**목적**: CRC 불일치 시 NACK 반환 및 재전송이 정상 동작하는지 검증

**절차 A — CRC 오류 주입**:
1. EC→STM32 송신 프레임의 CRC 바이트를 임의로 변조한다 (시리얼 인터셉터 또는 EC 디버그 모드 사용).
2. STM32 응답에서 NACK(Type=0x05) 반환 확인.
3. EC가 재전송을 수행하는지 확인.

**절차 B — 재전송 한계 검증**:
1. STM32 응답을 3초 이상 차단한다.
2. EC 로그에서 재전송이 최대 3회 수행되는지 확인.
3. 3회 후 통신 오류 처리 로그 확인.

**절차 C — 시퀀스 번호 중복 처리**:
1. 동일 SEQ 번호를 가진 프레임을 연속 전송한다.
2. STM32가 중복 메시지를 무시하는지 확인.

**합격 기준**:
- CRC 불일치: NACK 반환 (FR-012)
- 타임아웃(1초) 후 재전송 시작 (FR-014)
- 재전송 최대 3회 후 오류 처리 (FR-015)
- 중복 SEQ 무시 (FR-016)

**확인 사항**:
- [ ] NACK 정상 반환
- [ ] 재전송 3회 후 오류 처리
- [ ] 중복 시퀀스 무시

- 응답 누락 없음

**확인 사항**:
- [ ] 5초 주기 Heartbeat 정상 교환
- [ ] 10분간 누락 0건

## TC-FW-007 FreeRTOS 태스크 구조 및 독립성

| 항목 | 내용 |
|------|------|
| **추적** | FR-018, FR-019, FR-020 / NFR-009, NFR-011 / AC-013, AC-016 |
| **우선순위** | P1 |
| **분류** | 시스템 |

**목적**: 단일 태스크 장애가 다른 태스크에 영향을 주지 않는지 검증

**절차 A — TaskSensor 정지 시 독립성**:
1. STM32 디버거로 TaskSensor를 강제 정지(vTaskSuspend)한다.
2. EC 로그에서 TaskComm, TaskButton, TaskActuator 동작 지속 확인.
3. EC에서 센서 오류 알람(ALID-3) 수신 확인.
4. TaskSensor를 재개(vTaskResume)하고 정상 복구 확인.

**절차 B — TaskComm 정지 시 태스크 재시작**:
1. TaskComm을 강제 정지한다.
2. TaskWatchdog이 3초 내 감지하고 TaskComm을 재시작하는지 확인.
3. RESTART_REASON(0x03) 메시지 EC 수신 확인.

**절차 C — TaskSensor 수집 jitter 측정**:
1. 1시간 연속 동작 중 EC 수신 SENSOR_DATA 타임스탬프를 수집한다.
2. 연속 수신 간격의 표준편차와 최대 편차를 계산한다.

**합격 기준**:
- TaskSensor 정지 시 나머지 3 태스크 동작 유지 (FR-018, NFR-011)
- EC에서 ALID-3 알람 수신 (FR-025)
- TaskComm 재시작 시 RESTART_REASON 전송 (FR-024)
- TaskSensor jitter: **±50ms 이내** (NFR-009)

**확인 사항**:
- [ ] TaskSensor 정지 → 나머지 태스크 정상
- [ ] ALID-3 알람 발생
- [ ] TaskComm 재시작 정상
- [ ] jitter ±50ms 이내 (1시간 로그)

## TC-FW-008 하드웨어 Watchdog (IWDG) 자동 복구

| 항목 | 내용 |
|------|------|
| **추적** | FR-021, FR-022, FR-023, FR-024 / NFR-010 / AC-014 |
| **우선순위** | P1 |
| **분류** | 시스템 |

**목적**: 펌웨어 hang 발생 시 IWDG가 MCU를 재시작하고 EC와 재연결하는지 검증

**절차**:
1. STM32 펌웨어에 무한루프를 주입한다 (디버거 breakpoint 또는 테스트 빌드 플래그).
2. 스톱워치를 시작한다.
3. EC 로그에서 STM32 연결 해제(CEID-12) 감지 확인.
4. STM32 자동 재시작 후 EC 재연결 이벤트(CEID-11) 확인.
5. Host 로그에서 CEID-13(SystemRecovery) 이벤트 수신 확인.
6. 전체 소요 시간 측정.

**합격 기준**:
- MCU 재시작 → EC 재연결 → CEID-13 수신: **15초 이내** (NFR-010)
- RESTART_REASON 메시지에 재시작 원인 코드 포함 (FR-024)
- 재연결 후 정상 센서 데이터 수신 재개 (FR-023)

**확인 사항**:
- [ ] 15초 이내 완전 복구
- [ ] CEID-13 Host 수신
- [ ] 재시작 원인 코드 정상

## TC-FW-009 Graceful Degradation — 통신 장애 시 안전 상태

| 항목 | 내용 |
|------|------|
| **추적** | FR-026, FR-027 / NFR-011 |
| **우선순위** | P2 |
| **분류** | 시스템 |

**목적**: EC와의 UART 통신 중단 시 STM32가 안전 상태로 전환하는지 검증

**절차**:
1. STM32가 RUN 상태(팬 ON)에서 USB를 분리한다.
2. STM32의 팬/부저/LED 상태를 관찰한다.
3. 30초 후 USB를 재연결하고 재연결 정상 복구 확인.

**합격 기준**:
- 통신 중단 후 사전 정의 안전 상태 전환: 팬 OFF, 부저 ON, LED ALARM (FR-026)
- TaskSensor, TaskActuator는 계속 동작 (FR-027)
- 재연결 후 **10초 이내** 데이터 수신 재개 (NFR-006)

**확인 사항**:
- [ ] 안전 상태 전환 동작
- [ ] 재연결 10초 이내 복구

## TC-EC-002 Equipment State 전이 — 정상 흐름

5. 전체 소요 시간 측정.

**합격 기준**:
