# 1. 개요

## 1.1 목적

본 문서는 가상 공정 챔버에서 발생하는 주요 장애 시나리오를 재현하고,  
알람 발생부터 자동 안전 대응, 수동 원인 확인, 복구 검증, PM 수행까지  
현장 대응 절차(Field Response Procedure)를 구현하는 시스템의 소프트웨어 요구사항을 정의한다.

본 프로젝트는 반도체 장비 CS 엔지니어 직무에서 요구되는 다음 핵심 역량을 실습 대상으로 한다.

- 장애 유형별 원인 분류 (Fault Classification)
- 자동 안전 대응 (Automatic Safety Response)
- 복구 검증 후 재가동 승인 (Recovery Verification)
- 이상 징후 조기 감지 (Early Warning / Predictive Maintenance)
- 현장 대응 절차 문서화 (Troubleshooting Procedure)

## 1.2 범위

### 포함 범위
- 장애 시나리오 5종 재현: 과온, 도어 오픈, 히터 전류 이상, 센서 오류, 통신 단선
- 시나리오별 자동 안전 대응: 인터락 차단, 팬 가동, 부저 경보, SECS/GEM 알람 보고
- 수동 원인 확인: Host 화면에서 알람 코드 및 진단 결과 표시
- 복구 검증: 재가동 전 장비 상태 조건 체크 및 재가동 거부/승인
- 이상 징후 조기 감지: 히터 성능 저하 추이 감지 (온도 상승 속도 기반)
- PM 수행: 복구 완료 후 PM 점검 항목 체크 및 카운터 초기화

### 제외 범위
- 실제 반도체 공정 제어
- 다중 장비 클러스터 관리
- MES/ERP 연동
- 사용자 계정 및 권한 관리

## 1.3 시스템 목표

1. 장애 시나리오 6종을 실제 하드웨어에서 재현한다.
2. 장애 발생 시 자동으로 안전 대응 조치를 수행한다.
3. 동일한 장애 증상(전류 소실 등)에서 원인을 분류하여 진단 결과를 제공한다.
4. 복구 조건이 충족되지 않으면 재가동을 거부한다.
5. 히터 성능 저하를 장애 발생 전 사전에 감지한다.
6. 시나리오별 대응 절차를 Host 화면에서 운영자에게 안내한다.

## 1.4 문서 규칙

### 요구사항 키워드 (IEEE 29148-2018 준거)
- **shall**: 반드시 구현되어야 하는 요구사항 (검증 필수)
- **should**: 구현 권장 요구사항
- **may**: 선택 구현 요구사항

# 2. 현장 대응 절차 개요

## 2.1 공통 대응 흐름

모든 장애 시나리오는 아래 흐름을 따른다.

```
[1단계] 장애 발생
    │   센서 데이터 이상 감지 또는 통신 이벤트 수신
    ▼
[2단계] 자동 안전 대응  ← EC 자동 수행
    │   인터락 차단 (릴레이 OFF)
    │   팬 ON, 부저 ON
    │   SECS/GEM S5F1 알람 보고
    │   장애 이력 기록
    ▼
[3단계] 수동 원인 확인  ← 운영자 수행
    │   Host 화면에서 ALID + 진단 결과 확인
    │   원인 특정 (센서 데이터 / 전류 패턴 기반)
    ▼
[4단계] 복구 조치  ← 운영자 수행
    │   원인 제거 (냉각 대기, 도어 닫기, 배선 점검 등)
    │   ACK_ALARM + RESET 명령 입력
    ▼
[5단계] 복구 검증  ← EC 자동 수행
    │   재가동 조건 체크
    │   조건 미충족 시 재가동 거부 + 사유 표시
    ▼
[6단계] 재가동 승인
        IDLE 상태 전이 → 정상 운영 재개
```

## 2.2 장애 시나리오 목록

| 시나리오 ID | 장애 유형 | ALID | 인터락 여부 |
|------------|----------|------|------------|
| SC-01 | 과온 (Over-Temperature) | 1, 2 | 소프트웨어 + 하드웨어 이중 |
| SC-02 | 도어 오픈 (Door Open) | 3 | 소프트웨어 |
| SC-03 | 미가열 (Heater No-Heat) | 8 | 소프트웨어 |
| SC-04 | 히터 전류 이상 (Heater Current Anomaly) | 4 | 소프트웨어 |
| SC-05 | 센서 오류 (Sensor Error) | 5 | 소프트웨어 |
| SC-06 | 통신 단선 (Communication Disconnect) | 6 | 소프트웨어 |

# 3. 장애 시나리오별 요구사항

## 3.1 SC-01: 과온 시나리오

### 3.1.1 장애 재현 조건
챔버를 밀폐한 상태에서 히터를 장시간 가동하여 온도를 65°C 이상으로 상승시킨다.

### 3.1.2 자동 안전 대응 요구사항

**FR-SC01-01**  
시스템은 챔버 온도가 65°C를 초과하면 3초 이내에 ALARM 상태로 전이하고  
릴레이 OFF 명령, 팬 ON 명령, 부저 ON 명령을 STM32에 전송해야 한다.  
_AC: 온도 65°C 초과 시점부터 릴레이 OFF 수신까지 3초 이내_

**FR-SC01-02**  
시스템은 챔버 온도가 68°C를 초과하면 INTERLOCK 상태로 전이하고  
SECS/GEM S5F1 (ALID=2, AlarmSet=True)를 Host에 전송해야 한다.  
_AC: S5F1 메시지 수신 확인, ALID=2_

**FR-SC01-03**  
시스템은 ST-22 동작에 의한 전류 소실(INA219 전류 < 50mA, 릴레이 ON 상태)을  
감지하면 INTERLOCK 상태로 전이하고 ALID=4 알람을 발생시켜야 한다.  
ST-22는 자동복귀형이므로 챔버 냉각 후 전류가 자동 복귀될 수 있으나,  
EC는 운영자 RESET 명령 없이 자동으로 INTERLOCK을 해제하지 않는다.  
_AC: 전류 소실 감지 후 5초 이내 INTERLOCK 전이_

### 3.1.3 원인 분류 요구사항

**FR-SC01-04**  
EC는 과온 관련 전류 소실 발생 시 다음 기준으로 원인을 분류하고  
Host에 진단 결과를 표시해야 한다.

| 조건 | 진단 결과 |
|------|----------|
| 릴레이 ON + 전류 소실 + 온도 ≥ 65°C | ST-22 과온 차단 (70°C 동작, 자동복귀형) |
| 릴레이 ON + 전류 소실 + 온도 < 40°C | 히터 단선 / 배선 불량 |
| 릴레이 OFF 명령 직후 전류 소실 | 정상 동작 |

_AC: 각 조건에서 Host 화면에 해당 진단 결과 문자열 표시_

### 3.1.4 복구 검증 요구사항

**FR-SC01-05**  
시스템은 RESET 명령 수신 시 다음 조건을 모두 충족할 때만 IDLE 전이를 허용해야 한다.  
조건 미충족 시 재가동을 거부하고 미충족 항목을 Host에 표시해야 한다.

- 챔버 온도 < 50°C
- INA219 전류 정상 범위 (히터 회로 도통 확인)
- 도어 닫힘 상태

_AC: 온도 55°C에서 RESET 명령 입력 시 재가동 거부 및 사유 표시_

---

## 3.2 SC-02: 도어 오픈 시나리오

### 3.2.1 장애 재현 조건
챔버 가동 중 도어를 열어 리미트 스위치를 해제한다.

### 3.2.2 자동 안전 대응 요구사항

**FR-SC02-01**  
시스템은 도어 오픈 신호(PA1 LOW) 수신 후 1초 이내에 INTERLOCK 상태로 전이하고  
릴레이 OFF, 부저 ON 명령을 전송해야 한다.  
_AC: 도어 신호 수신 후 1초 이내 릴레이 OFF 수신_

**FR-SC02-02**  
시스템은 도어 오픈 인터락 발생 시 SECS/GEM S5F1 (ALID=3, AlarmSet=True)를  
Host에 전송해야 한다.  
_AC: S5F1 메시지 ALID=3 수신 확인_

### 3.2.3 복구 검증 요구사항

**FR-SC02-03**  
시스템은 RESET 명령 수신 시 도어 닫힘 상태가 확인되지 않으면  
재가동을 거부하고 "도어 닫힘 확인 필요" 메시지를 Host에 표시해야 한다.  
_AC: 도어 열린 상태에서 RESET 시 거부 메시지 표시_

---

## 3.3 SC-03: 미가열 시나리오

### 3.3.1 장애 재현 조건
START 명령 후 릴레이 ON 상태에서 일정 시간 동안 온도 상승이 없는 상황을 재현한다.  
히터 배선 탈거, 12V 전원 미인가, 릴레이 고장 중 하나를 선택하여 재현한다.

### 3.3.2 자동 안전 대응 요구사항

**FR-SC03-01**  
시스템은 릴레이 ON 명령 전송 후 30초 이내에 챔버 온도가 2°C 이상 상승하지 않으면  
ALARM 상태로 전이하고 ALID=8 (NO_HEAT) 알람을 발생시켜야 한다.  
_AC: 히터 배선 탈거 후 START 시 30초 이내 ALID=8 발생_

**FR-SC03-02**  
시스템은 미가열 감지 시 릴레이 OFF 명령을 전송해야 한다.  
_AC: ALID=8 발생 후 5초 이내 릴레이 OFF 수신_

### 3.3.3 원인 분류 요구사항

**FR-SC03-03**  
EC는 미가열 감지 시 INA219 전류 측정값과 온도 변화를 기준으로 원인을 분류하고  
Host에 진단 결과와 조치 안내를 표시해야 한다.

| INA219 전류 | 온도 변화 | 진단 결과 | 조치 안내 |
|------------|----------|----------|---------|
| 0mA | 없음 | 히터 회로 단선 (히터 소손 또는 배선 탈거) | 히터 배선 및 커넥터 점검 |
| 0mA | 없음 (릴레이 ACK 수신됐으나 전류 0) | 릴레이 미동작 (접점 불량) | 릴레이 모듈 점검 |
| 정격 30~70% | 없음 | 히터 성능 저하 (부분 단선) | PM 수행 — 히터 교체 검토 |
| 정상 범위 | 없음 | 챔버 누기 또는 히터 위치 이탈 | 챔버 밀폐 및 히터 접촉 점검 |

_AC: 각 조건에서 Host 화면에 해당 진단 결과 및 조치 안내 표시_

**FR-SC03-04**  
EC는 미가열 진단 결과와 함께 명령-실행 체인 추적 결과를 Host에 표시해야 한다.

```
[명령-실행 체인 추적]
  ✓ EC → StateMachine: START 명령 처리
  ✓ EC → STM32: 릴레이 ON 명령 전송
  ✓ STM32: 릴레이 ON 명령 ACK 수신
  ✗ INA219: 전류 미감지 (0mA)
  ✗ SHT31: 온도 상승 없음 (+0.1°C / 30s)
→ 진단: 히터 회로 단선 — 히터 배선 및 커넥터 점검
```

_AC: 미가열 발생 시 Host 화면에 체인 추적 결과 및 진단 문자열 표시_

### 3.3.4 복구 검증 요구사항

**FR-SC03-05**  
시스템은 RESET 명령 수신 시 다음 절차로 히터 회로 도통을 확인한 후 재가동을 허용해야 한다.  
조건 미충족 시 재가동을 거부하고 미충족 항목을 Host에 표시해야 한다.

1. 릴레이 ON 명령 전송 (테스트 펄스, 5초)
2. INA219 전류 정격 범위(500mA ~ 1200mA) 확인
3. 5초 이내 온도 상승 감지 (0.5°C 이상)
4. 조건 충족 시 IDLE 전이 허용

_AC: 히터 재연결 후 RESET 시 테스트 펄스에서 전류·온도 조건 확인 후 IDLE 전이_

---

## 3.4 SC-04: 히터 전류 이상 시나리오

### 3.4.1 장애 재현 조건
히터 배선을 탈거하여 전류 소실 상태를 만들거나,  
히터 가동 중 온도 상승 속도를 관찰하여 성능 저하를 감지한다.

### 3.4.2 자동 안전 대응 요구사항

**FR-SC04-01**  
시스템은 릴레이 ON 상태에서 INA219 측정 전류가 히터 정격 전류의 30% 미만  
(< 250mA, 정격 833mA 기준)이면 ALARM 상태로 전이하고 ALID=4 알람을 발생시켜야 한다.  
_AC: 히터 배선 탈거 후 30초 이내 ALID=4 알람 발생_

**FR-SC04-02**  
시스템은 히터 전류 이상 감지 시 릴레이 OFF 명령을 전송해야 한다.  
_AC: ALID=4 발생 후 5초 이내 릴레이 OFF 수신_

### 3.4.3 이상 징후 조기 감지 요구사항

**FR-SC04-03**  
시스템은 최근 10사이클 동안의 히터 가동 시 온도 상승 속도 평균을  
기준값(초기 10사이클 평균) 대비 지속적으로 추적해야 한다.  
_AC: Host 화면에 온도 상승 속도 추이 표시_

**FR-SC04-04**  
시스템은 온도 상승 속도가 기준값 대비 25% 이상 저하되면  
ALID=7 (PM_DUE) 알람을 발생시키고 "히터 성능 저하 감지" 메시지를 표시해야 한다.  
_AC: 상승 속도 저하 조건 충족 시 ALID=7 발생_

### 3.4.4 복구 검증 요구사항

**FR-SC04-05**  
시스템은 RESET 명령 수신 시 INA219 전류가 정격 범위(500mA ~ 1200mA)에 있을 때만  
재가동을 허용해야 한다.  
_AC: 배선 탈거 상태에서 RESET 시 재가동 거부_

## 3.5 SC-05: 센서 오류 시나리오

### 3.5.1 장애 재현 조건
SHT31 I2C 배선을 탈거하여 센서 응답 없음 상태를 만든다.

### 3.5.2 자동 안전 대응 요구사항

**FR-SC05-01**  
시스템은 SHT31 I2C 읽기 실패가 연속 3회 발생하면 ALARM 상태로 전이하고  
ALID=5 (SENSOR_ERROR) 알람을 발생시켜야 한다.  
_AC: I2C 탈거 후 5초 이내 ALID=5 발생_

**FR-SC05-02**  
시스템은 센서 오류 발생 시 안전을 위해 릴레이 OFF 명령을 전송해야 한다.  
온도 정보 없이 히터를 계속 가동하는 것은 허용하지 않는다.  
_AC: ALID=5 발생과 동시에 릴레이 OFF 수신_

**FR-SC05-03**  
시스템은 센서 오류 발생 전 마지막 정상 측정값과 오류 발생 시각을  
Host 화면에 표시해야 한다.  
_AC: Host 화면에 "마지막 정상 측정: XX°C (HH:MM:SS)" 표시_

### 3.5.3 복구 검증 요구사항

**FR-SC05-04**  
시스템은 RESET 명령 수신 시 SHT31 연속 3회 정상 읽기가 확인된 후에만  
재가동을 허용해야 한다.  
_AC: I2C 재연결 후 RESET 시 3회 읽기 대기 후 IDLE 전이_

---

## 3.6 SC-06: 통신 단선 시나리오

### 3.6.1 장애 재현 조건
STM32와 EC 사이의 USB-UART 케이블을 탈거하거나,  
STM32가 Heartbeat 전송을 중단하는 상황을 재현한다.

### 3.6.2 자동 안전 대응 요구사항

**FR-SC06-01**  
EC는 STM32로부터 Heartbeat가 10초 이상 수신되지 않으면  
ALARM 상태로 전이하고 ALID=6 (COMM_ERROR) 알람을 발생시켜야 한다.  
_AC: Heartbeat 미수신 10초 후 ALID=6 발생_

**FR-SC06-02**  
EC는 통신 단선 감지 시 릴레이 상태를 알 수 없으므로  
마지막으로 전송한 릴레이 명령 상태를 Host에 표시하고  
운영자에게 직접 확인을 요구해야 한다.  
_AC: Host 화면에 "릴레이 상태 불명 — 직접 확인 필요" 표시_

**FR-SC06-03**  
EC는 통신 단선 상태에서 Host로부터 START, STOP 등 장비 제어 명령을 수신하면  
명령을 거부하고 거부 사유를 Host에 표시해야 한다.  
_AC: 통신 단선 중 START 명령 시 거부 메시지 수신_

### 3.6.3 복구 검증 요구사항

**FR-SC06-04**  
시스템은 통신 재연결 후 Heartbeat 정상 수신이 확인되고  
STM32로부터 센서 데이터가 1회 이상 수신된 후에만 RESET을 허용해야 한다.  
_AC: 케이블 재연결 후 자동으로 통신 복구 감지 및 RESET 허용_

# 4. 공통 기능 요구사항

## 4.1 장애 이력 관리

**FR-HIST-01**  
시스템은 장애 발생 시각, ALID, 진단 결과, 복구 완료 시각, 복구 소요 시간을  
장애 이력으로 기록하고 Host 화면에 표시해야 한다.  
_AC: 장애 발생/복구 이벤트마다 이력 항목 추가 확인_

**FR-HIST-02**  
Host는 최근 20건의 장애 이력을 시간 역순으로 표시해야 한다.  
_AC: 21번째 이력 발생 시 가장 오래된 항목 삭제 확인_

## 4.2 복구 검증 공통 요구사항

**FR-RV-01**  
시스템은 RESET 명령 수신 시 복구 조건 체크 결과를 항목별로 Host에 표시해야 한다.

```
[복구 조건 확인]
  ✓ 온도 정상 (45°C < 50°C 기준)
  ✗ 히터 전류 이상 (측정값: 0mA, 기준: 500mA 이상)
  ✓ 도어 닫힘
→ 재가동 거부: 히터 전류 점검 필요
```

_AC: 조건별 합격/불합격 표시 및 거부 사유 항목 표시_

**FR-RV-02**  
시스템은 복구 조건이 모두 충족된 후 IDLE 전이 시  
SECS/GEM S6F11 (CEID=1, STATE_IDLE) 이벤트를 Host에 전송해야 한다.  
_AC: IDLE 전이 시 S6F11 CEID=1 수신 확인_

## 4.3 운영자 대응 안내

**FR-GUIDE-01**  
Host는 장애 발생 시 시나리오별 1차 조치 안내를 화면에 표시해야 한다.

| ALID | 표시 안내 |
|------|----------|
| 1, 2 | 챔버 냉각 대기 — 온도 50°C 미만 확인 후 RESET |
| 3 | 도어 닫힘 확인 후 RESET |
| 4 | 히터 배선 및 전원 점검 후 RESET |
| 5 | SHT31 I2C 배선 점검 후 RESET |
| 6 | UART 케이블 연결 확인 후 대기 (자동 복구) |
| 7 | PM 점검 수행 후 PM_RESET 명령 입력 |
| 8 | 히터 배선·커넥터·12V 전원·릴레이 모듈 순서로 점검 후 RESET |

_AC: 각 ALID 발생 시 해당 안내 문자열 Host 화면 표시_

## 4.4 SECS/GEM 보고

**FR-GEM-01**  
시스템은 장애 시나리오별 ALID에 대응하는 S5F1 알람 보고를 Host에 전송해야 한다.  
_AC: 각 시나리오 실행 시 해당 ALID의 S5F1 수신 확인_

**FR-GEM-02**  
시스템은 장비 상태 전이마다 S6F11 이벤트 보고를 Host에 전송해야 한다.  
_AC: IDLE/ALARM/INTERLOCK 전이 시 각 CEID의 S6F11 수신 확인_

# 5. 이상 징후 조기 감지 요구사항

## 5.1 온도 상승 속도 추적

**FR-EW-01**  
시스템은 히터 ON 시점부터 설정 온도(55°C)에 도달할 때까지의  
온도 상승 속도(°C/s)를 사이클마다 기록해야 한다.  
_AC: 각 사이클 완료 후 상승 속도 값 기록 확인_

**FR-EW-02**  
시스템은 초기 5사이클의 평균 상승 속도를 기준값으로 설정하고  
이후 사이클의 속도를 기준값과 비교하여 Host에 추이를 표시해야 한다.  
_AC: 5사이클 이후 Host 화면에 추이 표시 확인_

**FR-EW-03**  
시스템은 최근 3사이클 평균 상승 속도가 기준값 대비 25% 이상 저하되면  
"히터 성능 저하 감지" 경고를 Host에 표시하고 ALID=7을 발생시켜야 한다.  
_AC: 속도 저하 조건 충족 시 경고 및 ALID=7 발생_

---

# 6. PM 요구사항

## 6.1 PM 트리거 조건

**FR-PM-01**  
시스템은 다음 조건 중 하나라도 충족되면 ALID=7 (PM_DUE) 알람을 발생시켜야 한다.

| 조건 | 기준값 | 근거 |
|------|--------|------|
| 히터 누적 가동 시간 | 50시간 초과 | 카프톤 히터 절연 저하 이력 |
| 히터 사이클 카운트 | 200회 초과 | 릴레이 접점 마모 주기 |
| 온도 상승 속도 저하 | 기준 대비 25% 이상 | 히터 성능 저하 조기 감지 |

_AC: 각 조건 충족 시 ALID=7 발생 확인_

## 6.2 PM 점검 항목

**FR-PM-02**  
Host는 ALID=7 발생 시 PM 점검 항목을 체크리스트 형식으로 표시해야 한다.

```
[PM 점검 항목]
  □ 히터 저항 측정 (INA219 V/I — 정상: 12~16Ω)
  □ 릴레이 접점 저항 측정 (정상: < 0.5Ω)
  □ 온도 상승 속도 확인 (기준값의 75% 이상)
  □ 챔버 밀폐 상태 점검
  □ 배선 및 커넥터 체결 확인
```

_AC: ALID=7 발생 시 Host 화면에 체크리스트 표시_

**FR-PM-03**  
운영자가 PM_RESET 명령을 입력하면 시스템은 누적 가동 시간과 사이클 카운터를  
초기화하고 ALID=7을 해제해야 한다.  
_AC: PM_RESET 명령 후 가동 시간·사이클 0 초기화 및 ALID=7 해제 확인_

# 7. 비기능 요구사항

## 7.1 응답 시간

**NFR-RT-01**  
장애 감지부터 릴레이 OFF 명령 전송까지 3초 이내여야 한다.  
_AC: 각 시나리오에서 3회 측정 평균 3초 이내_

**NFR-RT-02**  
도어 오픈 인터락 응답은 1초 이내여야 한다.  
_AC: 도어 신호 수신부터 릴레이 OFF까지 1초 이내_

## 7.2 안전성

**NFR-SA-01**  
복구 검증 조건을 모두 충족하기 전까지 릴레이 ON 명령은 전송되지 않아야 한다.  
_AC: 미충족 상태에서 START 명령 입력 시 릴레이 ON 명령 미발생 확인_

**NFR-SA-02**  
EC와 STM32 간 통신 단선 시 시스템은 자동으로 ALARM 상태로 전이해야 한다.  
정상 통신 없이 장비 제어 명령을 전달하는 것은 허용하지 않는다.  
_AC: 통신 단선 10초 후 자동 ALARM 전이 확인_

## 7.3 이력 보존

**NFR-LOG-01**  
EC 프로세스가 종료되기 전까지 장애 이력은 메모리에 보존되어야 한다.  
_AC: EC 재시작 없이 20건 이력 전체 조회 가능_
