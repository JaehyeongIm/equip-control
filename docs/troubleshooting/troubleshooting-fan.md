# 팬 작동 오류 트러블슈팅 정리

## 1. 문제 상황

STM32 NUCLEO-F411RE와 IRF520 MOSFET 모듈을 이용해 5V 팬을 제어하려고 했으나,  
펌웨어에서 GPIO를 On/Off 해도 팬이 회전하지 않는 문제가 발생했다.

---

## 2. 사용 부품

- MCU: **STM32 NUCLEO-F411RE**
- 팬: **Raspberry Pi Cooling Fan [SZH-AT002]**
  - 정격 전압: 5V
  - 정격 전류: 0.2A ± 10%
- MOSFET 모듈: **IRF520**
  - 제어단자: `SIG`, `VCC`, `GND`
  - 전원입력단자: `VIN`, `GND`
  - 부하출력단자: `V+`, `V-`
- 계측기: **멀티미터**

---

## 3. 초기 의심 원인

처음에는 아래 가능성을 모두 열어두고 확인했다.

1. 팬 자체 불량
2. MOSFET 배선 오류
3. 터미널 블록 접촉 불량
4. GPIO 코드 오류
5. IRF520의 3.3V 구동 한계

---

## 4. 단계별 점검 내용

### 4-1. 팬 단독 직결 테스트 (5V)

팬을 STM32 보드의 전원핀에 직접 연결해 동작 여부를 확인했다.

- 팬 `+` → STM32 `5V`
- 팬 `-` → STM32 `GND`

#### 결과
- 팬이 **아주 잘 회전함**

#### 해석
- 팬 자체는 정상
- 팬 불량 가능성은 낮음

---

### 4-2. 팬 단독 직결 테스트 (3.3V)

팬이 저전압에서도 도는지 추가 확인했다.

- 팬 `+` → STM32 `3.3V`
- 팬 `-` → STM32 `GND`

#### 결과
- 팬이 **회전함**

#### 해석
- 이 팬은 3.3V 직결에서도 동작 가능한 수준
- 따라서 "팬이 3.3V에서는 절대 못 돈다"는 가정은 성립하지 않음
- 이후 문제는 팬 자체보다 **MOSFET 스위칭 특성** 쪽으로 좁혀짐

---

### 4-3. 팬/배선 기본 도통 테스트

팬 연결 자체가 끊어진 것은 아닌지 멀티미터 도통 모드로 기본 점검했다.

#### 점검 목적
- 팬 전선 단선 여부 확인
- 전원선/접지선의 기본 연결 상태 확인
- 브레드보드 및 점퍼선 연결 이상 여부 확인

#### 결과
- 기본 도통은 정상

#### 해석
- 팬 전선 자체나 단순 점퍼선 단선 가능성은 낮음
- 이후 원인은 팬 자체보다는 MOSFET 모듈/제어부 쪽으로 더 의심됨

---

### 4-4. MOSFET 모듈 제어 전압별 테스트

#### 테스트 A: SIG에 5V 입력
- `SIG` → 5V
- `VCC` → 5V
- `GND` → GND
- `VIN` → 5V
- `VIN GND` → GND
- `V+`, `V-` → 팬 연결

#### 결과
- 팬이 **정상 회전**

#### 해석
- IRF520 모듈 자체가 완전히 고장난 것은 아님
- 5V 제어에서는 동작 가능

---

#### 테스트 B: SIG에 3.3V 입력
- `SIG` → 3.3V
- `VCC` → 3.3V
- `GND` → GND
- `VIN` → 5V
- `VIN GND` → GND
- `V+`, `V-` → 팬 연결

#### 결과
- 팬이 **회전하지 않음**
- 모듈의 `VCC` LED는 점등됨

#### 해석
- 제어부에 전원은 들어감
- 그러나 MOSFET이 팬을 구동할 만큼 충분히 턴온되지 않음
- 즉, **LED 점등 = 실제 부하 구동 성공** 은 아님

---

### 4-5. GPIO 펌웨어 토글 테스트

STM32 GPIO를 3초 주기로 On/Off 하도록 펌웨어를 작성해 테스트했다.

#### 관찰 결과
- MOSFET 모듈 `VCC` LED가 3초 주기로 깜빡이며 반응
- 하지만 팬은 회전하지 않음

#### 해석
- GPIO 토글 코드 자체는 동작하고 있음
- 즉, **펌웨어 문제는 아님**
- STM32의 3.3V GPIO 신호가 MOSFET을 충분히 켜지 못하는 문제로 해석 가능

---

### 4-6. 터미널 블록 접촉 불량 의심 및 멀티미터 점검

MOSFET 모듈의 팬 연결부와 전원 연결부가 터미널 블록 구조라서,  
해당 연결부 접촉 불량 가능성을 의심했다.

#### 의심 배경
- 팬/전원 연결부가 터미널 블록이라 체결 불량이 발생할 수 있음
- 전선 피복이 물렸거나, 구리선 일부만 닿아 있어도 문제가 생길 수 있음

#### 점검 항목
1. **도통 검사**
2. **전압 전달 검사**

---

### 4-7. 터미널 블록 도통 검사

전원 OFF 상태에서 멀티미터 도통 모드로 다음을 확인했다.

#### 점검 대상
- 팬 빨간선 ↔ `V+`
- 팬 검은선 ↔ `V-`
- 5V 전원선 ↔ `VIN`
- GND 전원선 ↔ `GND`

#### 결과
- **정상**

#### 해석
- 터미널 블록에 물린 전선은 전기적으로 이어져 있음
- 단순 접촉 불량 가능성은 낮아짐

---

### 4-8. 터미널 블록 전압 전달 검사

전원 ON 상태에서 실제 전압이 전달되는지 측정했다.

#### 점검 목적
- 도통만으로는 부족하므로, 실제 부하 전압이 전달되는지 확인
- 전원이 터미널 블록을 통해 정상적으로 들어오고 나가는지 확인

#### 점검 항목
- `VIN - GND` 사이 전압
- MOSFET On 상태에서 `V+ - V-` 사이 전압

#### 결과
- **정상**

#### 해석
- 입력 전원은 터미널 블록을 통해 정상 공급되고 있음
- 출력 측 전압 전달도 기본적으로 살아 있음
- 따라서 터미널 블록 자체가 주원인일 가능성은 낮음

---

## 5. 핵심 비교 결과

|---|---:|---|
| 팬 5V 직결 | 동작 | 팬 정상 |
| 팬 3.3V 직결 | 동작 | 팬은 저전압에서도 회전 가능 |
| 팬/배선 기본 도통 검사 | 정상 | 팬 전선 및 기본 배선 이상 가능성 낮음 |

| 터미널 블록 전압 검사 | 정상 | 전원 전달 경로 정상 |

## 6. 최종 원인 분석

### 최종 원인
**IRF520 MOSFET 모듈은 STM32 NUCLEO-F411RE의 3.3V GPIO 출력으로는 팬을 구동할 만큼 충분히 턴온되지 않았다.**

### 이유
- STM32 F411RE의 GPIO HIGH는 기본적으로 **3.3V 로직**
- IRF520은 **5V 게이트 구동에서는 동작**
- 그러나 **3.3V 게이트 구동에서는 팬을 돌릴 만큼 충분히 열리지 않음**

즉, 이번 문제는 다음이 아니다.

- 팬 불량
- 기본 배선 단선
- 터미널 블록 접촉 불량
- 펌웨어 토글 실패

이번 문제는 아래로 정리된다.

> **부품 선택 문제 / 구동 전압 부적합 문제**

---

## 7. 기술적 해석

팬이 3.3V 직결에서는 도는데,  
IRF520의 `SIG=3.3V`에서는 팬이 돌지 않는 이유는 다음과 같다.

### 팬 3.3V 직결
- 팬 양단에 직접 3.3V가 걸림
- 팬이 저전압에서도 기동 가능하면 회전 가능

### IRF520 + SIG 3.3V

IRF520 데이터시트 기준:

| 항목 | 값 |
|------|---|
| VGS(th) 게이트 문턱 전압 | 2.0 ~ 4.0V |
| RDS(on) 도통 저항 | 0.27Ω @ VGS = **10V** 기준 |
| RDS(on) @ VGS = 3.3V | 수십 Ω 이상 (비정격 구간) |

VGS = 3.3V는 문턱 전압 턱걸이 수준이라 MOSFET이 완전히 열리지 않는다.  
이 상태에서 RDS가 급증하면 다음과 같은 전압 강하가 발생한다.

```
팬 전류: 0.2A
RDS @ VGS = 3.3V: 약 30Ω (추정)
전압 강하: 0.2A × 30Ω = 6V

팬에 걸리는 전압: 5V - 6V ≈ 0V → 팬 기동 불가
```

반면 LED는 전류가 수 mA 수준이라 같은 RDS에서도 전압 강하가 작아 켜진다.  
이것이 VCC LED는 점등되지만 팬은 돌지 않는 이유다.

즉, 둘 다 "3.3V"라는 숫자가 등장하지만 실제 의미는 다르다.

- **직결 3.3V** = 팬이 직접 3.3V를 받음
- **MOSFET 3.3V 제어** = MOSFET 게이트만 3.3V를 받아 스위칭 성능이 부족함

---

## 8. 트러블슈팅 결론

이번 트러블슈팅을 통해 다음을 검증했다.

1. 팬 자체는 정상
2. 팬은 3.3V 직결에서도 동작 가능
3. 기본 배선 및 팬 전선 도통은 정상
4. IRF520 모듈은 5V 제어에서는 동작
5. IRF520 모듈은 3.3V 제어에서는 팬 구동 실패
6. STM32 GPIO 토글 코드는 정상
7. 터미널 블록 도통 및 전압 전달은 정상
8. 따라서 근본 원인은 **IRF520의 3.3V 구동 부적합**

---

## 9. 최종 결론

> **STM32 F411RE의 3.3V GPIO는 IRF520 MOSFET을 팬 구동에 필요한 수준으로 충분히 켜지 못했다.**
>
> 따라서 이 구성은 STM32 3.3V GPIO 직결용 팬 제어 회로로 부적합하다.

## 10. 개선 방향

### 권장 방안
- **IRF520 → IRLZ44NPBF 교체**

### 선택 근거

| 항목 | IRF520 | IRLZ44NPBF |
|------|--------|-----------|
| VGS(th) 최대 | 4.0V | **2.0V** |
| 3.3V GPIO 제어 | 불가 | **가능** |
| RDS(on) @ VGS=3.3V | 수십 Ω | ~60mΩ |
| 패키지 | TO-220 | TO-220 (핀 호환) |
| 코드 수정 | — | **불필요** |

### 권장 구조
- 팬 전원: **외부 5V**
- MCU 역할: **GPIO(PB4)로 MOSFET 게이트 제어**
- 게이트 저항: **기존 1kΩ 그대로 사용**
- GND: **공통 GND 연결**

### 비권장 방안
- IRF520을 계속 쓰기 위해 5V 게이트 드라이브 회로를 추가하는 방법  
  → 가능은 하지만 회로가 불필요하게 복잡해짐

---

## 11. 프로젝트 관점에서의 의미

이번 이슈는 단순 실패가 아니라, 아래를 검증한 의미 있는 트러블슈팅 사례다.

- 팬 불량과 제어부 문제를 분리해서 검증함
- 멀티미터 도통 테스트로 기본 배선 이상 여부를 먼저 배제함
- 터미널 블록 도통 및 전압 전달 검사로 접촉 문제를 배제함
- 전원 경로, 제어 신호, 부하 특성을 단계적으로 분리해 확인함
- 코드 문제와 하드웨어 구동 한계를 구분함
- 최종적으로 **부품 선택의 적합성 문제**로 원인을 좁힘

즉, 이번 사례는 다음처럼 정리 가능하다.

> **"팬 작동 실패 문제를 단순 배선 문제로 보지 않고, 팬 단독 구동, 팬/배선 도통 검사, MOSFET 제어 전압 변화, GPIO 토글, 터미널 블록 도통/전압 점검을 통해 단계적으로 분리 진단한 결과, IRF520의 3.3V 구동 부적합 문제로 원인을 특정했다."**
