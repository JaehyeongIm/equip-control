# INA219 스파크 및 STM32 보드 손상 사고 트러블슈팅 (수정본)

## 1. 사고 개요

히터 전류 측정 회로(INA219) 배선 점검 중, 전원이 인가된 상태에서 INA219의 고전류 측(VIN+ 또는 VIN-) 전압을 확인하려고 프로브를 접촉하는 순간 INA219 모듈 표면에서 스파크가 발생하고 연기가 났다.
이후 STM32 NUCLEO-F411RE의 동작 이상이 확인되었고, 추가 점검 결과 **보드 손상이 유력**한 상태로 판단되었다.

---

## 2. 사고 시점 회로 구성

### 전력 회로 (24V 경로)

```text
WANPTEK 24V(+) ──→ INA219 VIN+ ──[shunt]──→ VIN- ──→ 히터(+)
히터(-) ──→ MOSFET Drain ──→ Source ──→ GND 레일
WANPTEK 24V(-) ──→ GND 레일
STM32 GND ──→ GND 레일
```

### I2C 회로 (브레드보드)

브레드보드 1열을 SCL 공통선, 2열을 SDA 공통선으로 사용했으며, INA219와 SHT31이 동일 I2C 버스에 병렬로 연결된 구조였다.

```text
STM32 3.3V ──→ 4.7kΩ ──→ 브레드보드 1열 (SCL)
STM32 3.3V ──→ 4.7kΩ ──→ 브레드보드 2열 (SDA)

브레드보드 1열 (SCL) ──→ STM32 PB8
                     ──→ INA219 SCL
                     ──→ SHT31  SCL

브레드보드 2열 (SDA) ──→ STM32 PB9
                     ──→ INA219 SDA
                     ──→ SHT31  SDA

INA219 VCC ──→ STM32 3.3V
SHT31  VCC ──→ STM32 3.3V
INA219 GND ──→ GND 레일
SHT31  GND ──→ GND 레일
```

사고 당시 조건:

- WANPTEK 24V 전원 인가 상태
- 히터 테스트 중 전류가 기대대로 읽히지 않아 배선/전압 확인 시도
- INA219의 고전류 측(VIN+ 또는 VIN-)에 멀티미터 프로브 접촉 시도

---

## 3. 재현 조건 및 관측 사실

| 항목 | 내용 |
|------|------|
| 멀티미터 다이얼 | 전압 모드(V) **확정** |
| 빨간 리드 단자 위치 | VΩ 단자 **확정** |
| 검은 프로브 접촉 위치 | 브레드보드 GND 레일 (STM32 GND와 공통) **확정** |
| 빨간 프로브 접촉 위치 | VIN- **확정** (인접 SDA 동시 접촉 가능성 있음) |
| 전원 상태 | WANPTEK 24V 인가 중 |
| PSU 전류 제한 | **미설정** |
| 스파크 위치 | 프로브 팁보다 **INA219 보드 표면 위**에서 관측, VIN- 핀 근처 |

추가 관측:

- 사고 직후 INA219 모듈에서 연기 발생
- 이후 USB 재연결 시 NUCLEO 보드 LED 상태가 평소와 다르게 보임
- ST-LINK GDB server 실행 시 `Target no device found` / `No device found on target` 오류 발생
- 히터 테스트(TEST_MODE) 정상 수행 불가

## 4. 원인 분석

### 4-1. 정상적인 전압 측정만으로는 스파크 설명이 어려움

멀티미터가 **정상적인 전압 측정 상태**였다면 입력 저항은 보통 매우 크며,
24V 노드를 GND 기준으로 측정하더라도 흐르는 전류는 극히 작다.
따라서 **정상적인 전압 측정만으로 INA219 보드에서 스파크와 연기가 난 상황은 설명하기 어렵다.**

즉, 아래 시나리오 중 하나가 있었을 가능성이 높다.

1. 멀티미터가 실제로는 **도통/저항 모드**였다.
2. 다이얼은 V였더라도 빨간 리드가 **A/mA 단자**에 꽂혀 있었다.
3. 프로브가 접촉 순간 **인접 노드까지 함께 건드려 직접 단락**을 만들었다.
4. 프로브가 점퍼선 금속부/헤더/패드 사이를 브리지했다.

### 4-2. VIN+ / VIN- 중 어느 쪽을 찍었는지는 불확실함

사고 당시 빨간 프로브를

- VIN+ 열의 WANPTEK + 점퍼선에 댔는지,
- VIN- 열의 히터 + 리드선에 댔는지,

정확히 기억나지 않는다.

다만 **스파크와 연기가 INA219 보드 표면에서 발생했다는 점**을 고려하면,
두 후보 중에서는 **VIN- 쪽 접촉 시 단락 전류가 INA219 보드의 shunt/패턴을 관통했을 가능성**이 상대적으로 더 잘 맞는다.

그러나 이는 어디까지나 **추정**이며, VIN+ 쪽 접촉이나 프로브 미끄러짐 가능성도 배제할 수 없다.

### 4-3. 사고의 본질: 순간 단락

현재까지의 정황상 사고의 본질은 **측정 동작 중 순간 단락(short)** 이 발생한 것으로 보는 것이 가장 타당하다.

즉, 직접 원인은

- 잘못된 측정 모드/단자,
- 또는 프로브 접촉 실수,
- 또는 둘의 조합

중 하나일 가능성이 높다.

## 5. 보드 손상 검사 및 측정 결과

사고 이후, **모든 외부 점퍼선/센서/히터/24V 전원을 제거하고 NUCLEO-F411RE 보드만 USB로 연결한 상태**에서 보드 손상 여부를 점검하였다.

### 5-1. ST-LINK / 타깃 연결 상태

GDB server 실행 결과:

```text
Target no device found
Error in initializing ST-LINK device.
Reason: No device found on target.
```

이는 ST-LINK USB 쪽은 일부 동작하더라도, **타깃 MCU가 정상적으로 응답하지 못하는 상태**와 일치한다.

### 5-2. 전압 측정 결과 (USB만 연결)

| 측정 지점 | 측정값 | 해석 |
|-----------|--------|------|
| CN6-5 (+5V) ↔ GND | 약 5V | 5V 전원 레일은 살아 있음 |
| CN6-4 (+3.3V) ↔ GND | 약 0.6V | **비정상** — 3.3V 레일이 심하게 무너져 있음 |

### 5-3. 저항 측정 결과 (전원 완전 제거 후)

| 측정 지점 | 측정값 | 해석 |
|-----------|--------|------|
| CN6-5 (+5V) ↔ GND | 약 7 kΩ | 비교적 정상 범주의 높은 저항 |
| CN6-4 (+3.3V) ↔ GND | 약 1.5 Ω | **매우 낮음** — 3.3V 레일 거의 단락 수준 |

### 5-4. 측정 결과 해석

위 결과를 종합하면:

- 5V 레일은 살아 있다.
- 반면 3.3V 레일은 **0.6V까지 붕괴**되어 있다.
- 전원 제거 후에도 3.3V-GND 저항이 **약 1.5Ω**로 매우 낮다.

이는 **3.3V 도메인에 심각한 손상 또는 거의 단락에 가까운 고장**이 생겼다는 뜻이다.

가장 가능성 높은 고장 후보는 다음과 같다.

1. **STM32F411 타깃 MCU 내부 손상**
2. **온보드 3.3V 레귤레이터 손상**
3. **기타 온보드 3.3V 부품 손상**

중요한 점은, 현재 측정 결과만으로 **정확히 어느 칩이 고장났는지 단정할 수는 없지만**,
**보드가 손상되었을 가능성은 매우 높다**는 점이다.

## 6. 결론

### 6-1. INA219 모듈

- 스파크와 연기가 직접 발생했으므로 **손상된 것으로 판단**한다.
- 재사용보다 **교체가 타당**하다.

### 6-2. STM32 NUCLEO-F411RE 보드

다음 정황이 모두 일치한다.

- ST-LINK에서 타깃을 찾지 못함
- USB 입력 5V는 존재함
- 3.3V 레일이 0.6V로 붕괴함
- 3.3V-GND 저항이 1.5Ω 수준으로 매우 낮음

따라서 **STM32 NUCLEO-F411RE 보드는 손상되었을 가능성이 매우 높으며, 실사용 기준으로는 교체 판단이 합리적**이다.

정확한 고장 부품을 보드 레벨로 추적할 수는 있겠지만,
현재 상태에서는 **수리보다 교체가 현실적**이다.

## 7. 재발 방지 대책

1. **전압 측정 전 3중 확인**
   - 다이얼이 DC V인지
   - 빨간 리드가 VΩ 단자인지
   - 측정 지점이 맞는지

2. **도통/저항 모드는 반드시 전원 OFF 후 사용**
   - 전원이 살아 있는 24V 회로에서 도통/저항 측정 금지

3. **24V / 히터 / 고전류 경로에는 듀퐁 점퍼선 사용 최소화**
   - 듀퐁 점퍼선은 신호선(UART/I2C/GPIO)에는 적합할 수 있으나,
     전원/히터/고전류 경로에는 접촉 불량·노출·단락 위험이 크다.
   - 전원 및 부하 전류 경로는 더 굵고 확실한 배선과 터미널 방식 사용 권장

4. **프로브 접촉 방식 개선**
   - 가능하면 GND를 먼저 고정
   - 고전류 패드/점퍼선 금속부를 직접 찌르기보다, 더 안전한 테스트 포인트 사용
   - 노출 금속이 짧은 프로브 또는 후크형 프로브 사용 고려

5. **PSU 전류 제한은 반드시 설정 후 전원 인가**
   - 전류 제한 미설정 상태에서 단락 발생 시 PSU 정격 전류가 그대로 흘러 손상 규모가 크게 커진다.
   - 히터 회로는 최대 소비 전류(24V/22Ω ≈ 1.1A)를 계산하고, 여유를 약간 준 값(예: 1.5A)으로 제한을 설정한 뒤 전원을 인가한다.
   - 배선 점검·멀티미터 측정 등 작업 중에는 전류 제한을 더 낮게(0.5A 이하) 유지하는 것을 권장한다.

6. **전원 인가 전 최소 검증 루틴 수행**
   - 24V(+)/GND 단락 여부
   - 3.3V/GND 비정상 저항 여부
   - MOSFET Drain/Source/Gate 오배선 여부
   - INA219 VIN+/VIN-/VCC/SDA/SCL 연결 재확인

## 8. 후속 조치 권고

- **INA219 모듈 교체**
- **STM32 NUCLEO-F411RE 보드 교체 권장**
- 새 보드 사용 전, 외부 24V 회로와 로직 회로를 분리해서 단계별 검증
- 첫 전원 인가 시에는
  1. 보드 단독 USB 전원 확인
  2. I2C 센서만 연결
  3. MOSFET/히터 회로는 마지막에 연결
- 가능하면 전원 인가 초기에는 **전류 제한(current limit)** 을 낮춰서 검증

## 9. SHT31 온습도 센서 손상 확인 절차 (추가)

스파크 사고 이후 SHT31도 함께 손상되었을 가능성이 있어 단계별로 진단했다.

### Step 1 — 전압 계측 (멀티미터, 전원 인가 상태)

| 측정 지점 | 측정값 | 판정 |
|----------|--------|------|
| SHT31 VCC ↔ GND | 3.3V | 정상 |
| SCL 라인 ↔ GND | 3.3V | 정상 (풀업 동작) |
| SDA 라인 ↔ GND | 3.3V | 정상 (풀업 동작) |

→ 전원 및 풀업 배선 정상.

### Step 2 — I2C 핀 매핑 확인

CubeMX에서 I2C1 추가 시 기본값이 PB6/PB7로 잡혀 있었음. 센서는 PB8(D15)/PB9(D14)에 연결돼 있어 핀 불일치 발생. `stm32f4xx_hal_msp.c`에서 확인 후 PB8/PB9로 재설정.

### Step 3 — I2C 스캔

주소 0x01~0x7F 전체를 `HAL_I2C_IsDeviceReady()`로 스캔.

결과: **no device found** — 어떤 주소에서도 응답 없음.

### Step 4 — SCL 파형 확인

스캔 실행 중 SCL 핀 전압 측정 시 3.3V → 3.0V → 3.3V로 흔들림 확인.

→ MCU가 I2C 클럭을 정상 출력하고 있음. MCU 및 배선 이상 없음.

### 결론

MCU 정상, 배선 정상, 전압 정상, SCL 클럭 정상임에도 불구하고 응답 없음 → **SHT31 손상 확정**.

## 11. 문서 검토로 수정한 핵심 사항

기존 정리 문서에서 아래 부분은 수정/보완이 필요했다.

1. **I2C 핀 표기**
   - 기존 문서의 `PB7/PB8` 표기는, 현재 사용자 연결 기준과 일치하지 않았다.
   - 본 수정본에서는 사용자 정리 기준인 **PB8(SCL), PB9(SDA)** 로 반영했다.

2. **프로브 접촉 위치 단정 표현**
   - 기존 문서에는 `VIN- 열에 갖다댔다`고 단정한 문장이 있었으나,
     실제 대화 기준으로는 **VIN+ 또는 VIN- 중 불확실**하다.

3. **LED 해석 단정 표현**
   - 기존 문서의 `LD1 즉시 빨간불` 등은 보드의 실제 LED 의미와 사용자 관찰이 혼재되어 있어,
     수정본에서는 이를 **"평소와 다른 LED 상태"**로 보수적으로 정리했다.

4. **고장 부품 단정 표현**
   - 기존 문서는 `3.3V LDO 레귤레이터 파손 확정`으로 적었지만,
     현재 측정 결과만으로는 **MCU / 3.3V 레귤레이터 / 기타 3.3V 부품 중 정확한 고장 부위는 단정 불가**하다.
   - 따라서 수정본에서는 **"보드 손상이 유력"**으로 표현을 조정했다.

5. **5V가 살아 있는 이유 설명**
   - 기존 문서는 `5V는 INA219와 미연결이었으므로 손상 없음`으로 적었지만,
     더 정확한 표현은 **"USB 5V 전원 경로는 살아 있고, 손상이 3.3V 도메인에 집중되었을 가능성이 높다"**이다.

## 12. 추가 분석 — 최종 추정 원인

사고 이후 추가로 확인된 정보를 모두 반영하여 가장 유력한 손상 시나리오를 정리한다.

**최종 확인 사항:**
- INA219 실제 핀 배열: `VCC  GND  SCL  SDA  VIN-  VIN+`
- 멀티미터: 전압 모드(V) **확정**
- 검은 프로브: GND **확정**
- 빨간 프로브: VIN- **확정** (단, 인접한 SDA를 동시 접촉했을 가능성 있음)
- 스파크 위치: INA219 VIN- 핀 근처
- 테스트 코드 실행 여부: 실행 중이 아니었을 가능성 높음
- PSU 전류 제한: 미설정 상태

### 12-1. 핀 배열에서 드러나는 결정적 단서

```
VCC   GND   SCL   SDA   VIN-   VIN+
                   ↑      ↑
               바로 인접 (2.54mm 간격)
```

**SDA와 VIN-는 바로 옆 핀이다.** 전압 모드에서 빨간 프로브를 VIN-에 올바르게 접촉했더라도, 멀티미터 내부 저항이 10MΩ이라 흐르는 전류는 2.4µA에 불과하여 스파크가 발생할 수 없다. 따라서 프로브 팁이 VIN-에 닿는 순간 바로 옆 SDA 핀 또는 SDA에 꽂힌 점퍼선의 금속부를 동시에 건드렸을 가능성이 가장 유력하다.

### 12-2. 가장 유력한 원인 — 프로브에 의한 VIN-·SDA 브릿지

```
빨간 프로브가 VIN-(약 24V)에 닿으면서 인접한 SDA를 동시 접촉
  → 24V가 SDA 라인으로 직접 유입
  → 스파크·아크: VIN- 핀 근처 (프로브 접촉 지점)
  → PSU 전류 제한 미설정 → 에너지 무제한 공급
```

SDA 라인에는 INA219·SHT31·STM32 PB9가 모두 연결되어 있다.

```
SDA 라인에 24V 유입
        │
        ├─ INA219 SDA 핀 ESD 다이오드 도통 → INA219 VCC(3.3V)로 역류
        ├─ SHT31  SDA 핀 ESD 다이오드 도통 → SHT31  VCC(3.3V)로 역류
        └─ STM32  PB9  ESD 다이오드 도통 → STM32  3.3V 레일로 역류
                  │
                  ▼
        3.3V 레일 전체 과전압
                  │
                  ├─ INA219 소손 (스파크·연기 — 프로브 브릿지 지점)
                  ├─ STM32 3.3V 도메인 손상 (실측: 3.3V → 0.6V, 저항 1.5Ω)
                  └─ SHT31 VCC 손상 (I2C 스캔 무응답, SCL 클럭은 정상)
```

### 12-3. "INA219가 24V를 측정할 수 있다"는 "어떤 핀에든 24V가 닿아도 안전하다"는 뜻이 아님

이 사고를 이해할 때 흔히 생기는 혼동을 정리한다.

**VIN+/VIN-에 24V가 걸리는 것은 정상이다**

INA219는 버스 전압 0~26V 범위를 측정하도록 설계되어 있다. 정상 연결 시 각 핀의 전위는 다음과 같다.

```
VIN+ ≈ 24.0V
VIN- ≈ 23.9V  (VIN+ - VIN- = 션트 양단 전압 ≈ 0.1V)
```

이 상태는 INA219 정격 범위 내의 정상 동작이다.

**SDA에 24V가 닿는 것은 전혀 다른 문제다**

INA219 SDA 핀의 절대 최대 전압은 데이터시트 기준 GND 대비 약 -0.3V ~ +6V이다.

```
SDA 허용 최대 전압: 약 +6V
사고 시 유입 전압:   약 24V
초과분:             약 18V
```

24V 시스템 자체가 문제가 아니라, **24V가 들어가면 안 되는 SDA(I2C 로직 핀)로 유입된 것이 문제다.**

**"Max current 3.2A"도 SDA와 무관하다**

INA219 모듈 사양에 표기되는 Max current 3.2A는 션트저항 양단 전류 측정 범위를 뜻한다.

```
션트 측정 최대 전압: ±320mV
션트 저항:           0.1Ω
I = 0.32V / 0.1Ω = 3.2A
```

이는 VIN+/VIN- 사이를 정상적으로 흐르는 부하 전류를 최대 3.2A까지 측정할 수 있다는 의미다. SDA·SCL·STM32 GPIO·SHT31 I2C 핀은 mA 단위만 허용하는 로직 핀이며, 이 핀들에 24V가 유입되면 3.2A보다 훨씬 작은 전류에서도 파손된다.

### 12-4. 이 시나리오가 모든 관측 사실을 설명하는 이유

| 관측 사실 | 설명 |
|---------|------|
| 스파크가 VIN- 핀 근처에서 발생 | 프로브 브릿지 아크 발생 지점 |
| INA219 소손 | 아크 직격 + ESD 다이오드 과전류 |
| STM32 3.3V 레일 붕괴 | SDA → ESD 다이오드 → 3.3V 역류 |
| SHT31 손상 | SDA 공유 → 동일 경로로 동시 손상 |
| 테스트 코드 실행 불필요 | 프로브 접촉만으로 성립 |
| 3부품 동시 손상 | SDA 공유선 하나로 모두 연결 |

### 12-5. PSU 전류 제한 미설정의 역할

전류 제한이 설정되어 있었다면 SDA 라인으로 유입되는 전류가 억제되어 손상 규모가 줄었을 수 있다. **전류 제한 미설정은 사고의 직접 원인은 아니지만 손상을 INA219 단독 소손에서 STM32·SHT31 연쇄 손상으로 확대시킨 보조 요인**이다.

### 12-6. 구조적 취약점 요약

| 취약점 | 내용 |
|--------|------|
| SDA와 VIN- 핀 인접 배치 | 프로브 실수로 24V가 I2C 라인에 유입되는 경로 형성 |
| SDA 라인에 보호 소자 없음 | 24V 유입 시 연결된 모든 IC의 ESD 다이오드가 동시 도통 |
| PSU 전류 제한 미설정 | 유입 에너지 제한 없이 3.3V 레일 전체 손상 |

### 멀티미터 측정 전 체크리스트 (이 사고 이후 수립)

이 사고를 직접적인 계기로 아래 체크리스트를 수립하여 동일한 실수가 재발하지 않도록 한다. 24V 회로에서 멀티미터로 측정하기 전에 반드시 세 항목을 순서대로 확인한다.

| 순서 | 항목 | 확인 |
|------|------|------|
| 1 | **측정 대상 모듈의 핀 배열을 데이터시트로 확인** — 고전압 측 핀(VIN+/VIN-)과 로직 핀(SDA/SCL/GND)의 위치를 파악한다 | ☐ |
| 2 | **PSU 전류 제한을 정상 동작 전류의 1.3배 이하로 설정** — 단락 발생 시 유입 에너지를 제한한다 | ☐ |
| 3 | **전원을 OFF한 상태에서 후크형 클립 프로브를 측정 지점에 고정** — 프로브를 손으로 누르지 않아 미끄러짐에 의한 인접 핀 접촉을 방지한다. 고정 후 전원을 켜고 측정한다 | ☐ |
