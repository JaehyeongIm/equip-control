# 릴레이 모듈 팬 제어 오류 트러블슈팅

## 1. 문제 상황

STM32 NUCLEO-F411RE와 SZH-EK082 1채널 릴레이 모듈로 팬을 제어하려 했으나,
TEST_MODE 팬 테스트([5/5]) 실행 시 릴레이가 동작하지 않고 팬이 회전하지 않았다.

---

## 2. 사용 부품

- MCU: **STM32 NUCLEO-F411RE**
- 릴레이 모듈: **SZH-EK082 1채널 5V 미니 릴레이 모듈**
  - 제어 핀: S, +, -
  - 외부 단자: COM, NO, NC
- 팬: **Raspberry Pi Cooling Fan [SZH-AT002]** (5V, 0.2A)
- 제어 핀: PB4 (D5)

---

## 3. 초기 배선

| 연결 | 내용 |
|------|------|
| 릴레이 + | STM32 CN6-5 (5V) |
| 릴레이 - | GND 레일 |
| 릴레이 S | PB4 (D5) |
| COM | 5V |
| NO | 팬(+) |
| 팬(-) | GND 레일 |

---

## 4. 단계별 점검

### 4-1. 릴레이 코일 전원 확인

멀티미터로 릴레이 +, - 핀 사이 전압 측정 → **5V 정상**

### 4-2. PB4 신호 전압 확인

TEST_MODE 실행 중 PB4와 GND 사이 전압 측정:
- FAN ON 구간: **2.2V** (3.3V여야 함)
- FAN OFF 구간: **0.8V** (0V여야 함)

평상시(테스트 미실행)에도 PB4에서 2.3V 측정됨.

#### 해석
릴레이 모듈 S핀 내부 풀업이 5V 기준으로 설계되어 있어, STM32 GPIO 출력(3.3V)을 끌어올려 중간 전압(2.2V)에 머물게 함. 릴레이 ON에 필요한 충분한 전압을 확보하지 못한 상태.

### 4-3. S핀 GND 직결 테스트

S핀을 PB4에서 분리 후 GND 레일에 직접 연결 → 딸깍 소리 없음

#### 해석
LOW 트리거 방식도 아님. 신호 방식 문제가 아닌 전원 기준 전압 문제로 범위 좁힘.

### 4-4. S핀 5V 직결 테스트

S핀을 5V 레일에 직접 연결 → **딸깍 소리 발생, 팬 회전**

#### 해석
릴레이 모듈 자체는 정상. 5V 신호에서는 동작 확인. 제어 신호 전압 부족이 원인임을 확정.

---

## 5. 원인 분석

SZH-EK082는 5V 릴레이 모듈로, 내부 S핀 회로가 5V 기준으로 설계되어 있다.
STM32 GPIO HIGH(3.3V)는 5V 기준에서 불충분한 신호로 인식되어 릴레이가 ON되지 않는다.

---

## 6. 해결 방법

릴레이 모듈 전원을 5V에서 **3.3V로 변경**.

| 연결 | 변경 전 | 변경 후 |
|------|--------|--------|
| 릴레이 + | 5V (CN6-5) | **3.3V (CN6-4)** |
| COM | 5V | **3.3V** |

전원 기준이 3.3V로 낮아지면서 STM32 GPIO HIGH(3.3V)가 충분한 신호로 인식되어 릴레이 정상 동작.

---

## 7. 결론

> SZH-EK082를 STM32 3.3V GPIO로 제어할 경우, 릴레이 모듈 전원(+)과 COM을 3.3V로 공급해야 한다.
> 5V 공급 시 S핀 내부 풀업 기준이 높아져 3.3V GPIO 신호가 불충분하게 인식된다.
