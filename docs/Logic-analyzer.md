# 로직분석기 채널 매핑 문서

## 1. 목적
STM32 Nucleo-F411RE 기반 가상 챔버 프로젝트에서 UART, I2C, GPIO/PWM 신호를 로직분석기로 캡처하기 위한 채널 매핑을 정리한다.

---

## 2. 로직분석기 채널 매핑

| 채널 | 신호 | STM32 핀 | 목적 |
|---|---|---|---|
| CH1 | UART TX | PA2 | STM32 → EC 프레임 확인 |
| CH2 | UART RX | PA3 | EC → STM32 명령 확인 |
| CH3 | I2C SCL | PB8 | 센서 클럭 확인 |
| CH4 | I2C SDA | PB9 | 센서 데이터 확인 |
| CH5 | HEATER PWM | PB0 (A3) | 히터 PWM duty 확인 |
| CH6 | FAN GPIO | PB4 (D5) | 팬 ON/OFF 제어 신호 확인 |
| CH7 | BUZZER | PB10 (D6) | 부저 출력 신호 확인 |
| CH8 | DOOR SW | PA1 (A1) | 도어 스위치 상태 확인 |
| GND | GND | CN6-6 / CN6-7 | 로직분석기 기준 전위 연결 |

---

## 3. 신호별 확인 목적

### UART
- **CH1 / PA2 (UART TX)**  
  STM32가 EC로 전송하는 프레임이 정상적으로 출력되는지 확인한다.
- **CH2 / PA3 (UART RX)**  
  EC에서 STM32로 들어오는 명령 프레임이 정상적으로 수신되는지 확인한다.

### I2C
- **CH3 / PB8 (I2C SCL)**  
  I2C 클럭이 정상적으로 발생하는지 확인한다.
- **CH4 / PB9 (I2C SDA)**  
  센서 데이터 송수신 및 ACK/NACK 동작을 확인한다.

### 출력 제어 신호
- **CH5 / PB0 (HEATER PWM)**  
  히터 제어용 PWM duty 변화와 주기를 확인한다.
- **CH6 / PB4 (FAN GPIO)**  
  팬 ON/OFF 제어 신호가 의도한 시점에 출력되는지 확인한다.
- **CH7 / PB10 (BUZZER)**  
  경보 발생 시 부저 구동 신호가 출력되는지 확인한다.

### 입력 신호
- **CH8 / PA1 (DOOR SW)**  
  도어 열림/닫힘 상태 변화가 GPIO 입력에 정상 반영되는지 확인한다.

---

## 4. 연결 시 주의사항
- 로직분석기의 **GND는 반드시 STM32 보드 GND와 공통 접지**로 연결한다.
- UART, I2C, GPIO 신호는 모두 **3.3V 로직 기준**으로 측정한다.
- I2C 측정 시 SCL/SDA 두 채널을 함께 연결해야 버스 해석이 쉽다.
- PWM은 단순 HIGH/LOW뿐 아니라 **주기와 duty 비율**을 함께 확인한다.
- 도어 스위치 입력은 풀업/풀다운 설정에 따라 평상시 레벨이 달라질 수 있으므로, 펌웨어 설정과 함께 해석한다.

---

## 5. 권장 확인 순서
1. **GND 공통 연결 확인**
2. **UART TX/RX 파형 확인**
3. **I2C SCL/SDA 통신 확인**
4. **HEATER PWM 출력 확인**
5. **FAN / BUZZER GPIO 출력 확인**
6. **DOOR SW 입력 변화 확인**

---

## 6. 요약
이번 로직분석기 매핑은 다음 목적을 가진다.

- **UART**: STM32 ↔ EC 간 프레임 송수신 검증
- **I2C**: 센서 통신 검증
- **PWM/GPIO**: 히터, 팬, 부저 제어 신호 검증
- **입력 GPIO**: 도어 상태 입력 검증

이 매핑으로 통신, 제어, 입력 신호를 한 번에 캡처하여 디버깅할 수 있다.
