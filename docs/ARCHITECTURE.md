## 2. 계층별 역할

| 계층 | 플랫폼 | 핵심 역할 |
|------|--------|----------|
| **STM32 펌웨어** | NUCLEO-F411RE | 챔버 센서 수집, 히터 릴레이/팬/부저 제어, 도어 감지, EC와 UART 통신 |
| **EC 소프트웨어** | macOS (터미널, C++) | GEM 상태 머신, 이상 진단, 인터락 판단, PM 추적, 알람/이벤트 관리, HSMS 서버 |
| **Host 프로그램** | macOS (터미널, Python) | 챔버 상태/센서/PM 표시 (curses GUI), 원격 명령 전송 |

## 4. EC 소프트웨어 내부 구조

### 모듈 의존 관계

```
main.cpp
  ├── DeviceComm        ← STM32 UART 수신 콜백
  │     └── onFrame() ──► FaultDiagnosis.analyze()
  │                   ──► InterlockChecker
  │                   ──► AlarmManager.checkAndSet()
  │                   ──► StateMachine.processEvent()
  │                   ──► PMTracker.onHeater*()
  │                   ──► HsmsServer.sendEventReport()
  │
  ├── StateMachine      ← 상태 전이 시 콜백
  │     └── onStateChange() ──► DeviceComm.sendCommand()
  │                        ──► HsmsServer.sendEventReport()
  │
  ├── AlarmManager      ← 알람 발생/해제 시
  │     └── ──► HsmsServer.sendAlarmReport()
  │
  ├── PMTracker         ← 주기적 PM 상태 확인
  │     └── isPMDue() ──► AlarmManager.checkAndSet(PM_DUE)
  │
  └── HsmsServer        ← Host 명령 수신 시
        └── onRemoteCmd() ──► StateMachine.processEvent()
                         ──► AlarmManager.ack()
                         ──► PMTracker.reset()
```

## 6. PM 추적 흐름

```
히터 ON
  │
  ▼
PMTracker.onHeaterOn()
  타이머 시작

히터 OFF
  │
  ▼
PMTracker.onHeaterOff()
  누적 가동 시간 += 경과 시간
  사이클 카운트++

메인 루프 (1초마다)
  │
  ▼
PMTracker.isPMDue()?
  ├── YES ──► AlarmManager.checkAndSet(ALID_PM_DUE)
  │            HsmsServer.sendAlarmReport(7, true)
  └── NO  ──► 계속

PM_RESET 명령 수신
  │
  ▼
PMTracker.reset()
  누적 가동 시간 = 0
  사이클 카운트 = 0
  AlarmManager.tryClear(ALID_PM_DUE)
```
